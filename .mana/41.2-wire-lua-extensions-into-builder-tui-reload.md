Connect the imp-lua extension system to the rest of imp.

### Builder integration (imp-core/src/builder.rs)
1. In AgentBuilder::build(), after registering native tools:
   - Scan extension directories via imp_lua::loader::discover_extensions()
   - Create LuaRuntime, load extensions
   - Call imp_lua::bridge::load_lua_tools(runtime, &mut agent.tools)
2. Store the LuaRuntime in Agent or AgentHandle so it persists across calls

### TUI slash command dispatch (imp-tui/src/app.rs)  
1. Store LuaRuntime (or the command handles) in App
2. When user types a slash command not in the built-in list, check Lua commands
3. Call the Lua command handler, show output as system message

### /reload wiring
The /reload command already reloads config. Extend it to also:
1. Re-scan extension directories
2. Re-evaluate .lua files
3. Re-register tools and commands

### Trust store
1. Create ~/.config/imp/trusted_extensions.json
2. On first load of project extensions (.imp/extensions/), prompt user
3. Store approved project paths in the trust file
4. Skip prompt for user extensions (~/.config/imp/extensions/)

### Files
- `imp/crates/imp-core/src/builder.rs` — load Lua extensions
- `imp/crates/imp-tui/src/app.rs` — dispatch Lua commands, reload
- `imp/crates/imp-cli/src/main.rs` — pass extension runtime through

### Tests
- Test that a .lua extension in a temp dir registers as a tool
- Test /reload picks up new extensions
- Test unknown slash command routes to Lua handler

- ctx.tool bridge
