Merge `MultiEditTool` into `EditTool` so there's one `edit` tool that handles both cases.

Detection logic:
- If params has `edits` array → multi-edit mode (apply sequential edits)
- If params has `oldText` + `newText` → single edit mode (find and replace)
- Both require `path`

Keep the existing implementation in edit.rs and multi_edit.rs. The unified EditTool
checks which params are present and delegates to the right implementation.

Update registration in imp-cli/src/main.rs to only register `EditTool`.
