Add visibility into what agents are doing between Spawning and Completed events.

New PoolEvent variants:
- Progress { unit_id, phase, elapsed } — agent reports what it's doing
- Heartbeat { unit_id, elapsed } — agent is still alive

Extend the Spawner trait with an optional progress channel. Spawner implementations can send progress updates while the agent runs. The dispatch loop forwards them as PoolEvents.

Add a heartbeat timeout: if no progress or heartbeat arrives within idle_timeout_minutes, the pool can mark the agent as stuck (separate from killing it, which is the spawner's job).

New PoolEvent::AgentStuck { unit_id, last_progress_secs_ago }.
