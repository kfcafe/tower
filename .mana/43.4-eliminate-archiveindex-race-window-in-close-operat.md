## Problem

In `mana/crates/mana-core/src/ops/close.rs`, the `close()` function (line 585) calls `rebuild_index(mana_dir)` AFTER `archive_unit()` (line 582) which renames the unit file from its original path to `archive/`. The `archive_unit` function also appends to the archive index, but between the file rename and the index rebuild, any concurrent reader calling `get_unit()` → `find_unit_file()` will fail because:

1. The unit markdown file no longer exists at its original path
2. The main index still has the old entry (not yet rebuilt)
3. `find_unit_file()` uses glob on the filesystem, not the index

This race window also means that if `rebuild_index` fails (e.g., if another unit file is malformed), the unit has already been moved but the index is stale.

## Solution

Reorder the close operation to rebuild index immediately after marking the unit as archived, and ensure the archive happens atomically with the index update.

In `mana/crates/mana-core/src/ops/close.rs`, modify the close sequence (around lines 550-590):

### Current order:
```
1. unit.status = Closed
2. unit.to_file(&unit_path)
3. archive_unit()  ← moves the file
4. post-close actions
5. auto-close parents
6. rebuild_index()  ← stale window between 3 and 6
```

### New order:
```
1. unit.status = Closed
2. unit.to_file(&unit_path)
3. archive_unit()  ← moves the file
4. rebuild_index()  ← immediate, closes the stale window
5. post-close actions
6. auto-close parents
7. (rebuild_index already done, auto-close parents may need another rebuild)
```

Specifically, move the first `rebuild_index(mana_dir)?;` call to immediately after `archive_unit()`, before the post-close actions. The later `rebuild_index` call (after auto-close parents, line 585) can remain as-is since parent closing may modify more files.

This is a minimal change — just reordering existing code.

### In `close()` function body, find the section around lines 577-590:

Current:
```rust
// 6. Archive
let archive_path = archive_unit(mana_dir, &mut unit, &unit_path)?;

// 7. Post-close cascade
let post_close = run_post_close_actions(...);
warnings.extend(post_close.warnings);

// ... worktree cleanup ...

// 8. Auto-close parents
let auto_closed_parents = ...;

// Rebuild index before auto-commit
rebuild_index(mana_dir)?;
```

New:
```rust
// 6. Archive
let archive_path = archive_unit(mana_dir, &mut unit, &unit_path)?;

// 6b. Rebuild index immediately after archive so stale entries don't linger
rebuild_index(mana_dir)?;

// 7. Post-close cascade
let post_close = run_post_close_actions(...);
warnings.extend(post_close.warnings);

// ... worktree cleanup ...

// 8. Auto-close parents
let auto_closed_parents = ...;

// Rebuild index again after parent closures
rebuild_index(mana_dir)?;
```

## Don't
- Don't change archive_unit itself
- Don't change the archive index (ArchiveIndex) logic
- Don't change how find_unit_file works
- Don't add new functions
