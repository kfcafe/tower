Remove or sharply narrow the last remaining synchronous join-harvest style bridge in the imp interactive runtime supervision path. Earlier cleanup moved terminal ownership to a top-level runner, introduced supervised background tasks, and shifted much of the runtime into a central signal pump. However, there is still a remaining pattern where task completion is harvested through synchronous bridging semantics rather than a more fully async-native completion path. This unit is specifically about finishing that runtime-shape cleanup without regressing behavior. Work may include redesigning how startup-task completion, agent-task completion, login-task completion, or runtime signals are observed; tightening how the event loop consumes completion state; and ensuring that error visibility, cancellation semantics, and terminal restoration guarantees remain intact. The point is not to chase theoretical purity, but to reduce architectural friction: prompt submission should not depend on awkward sync/async bridging, runtime supervision should remain understandable, and task completion should flow through a clearly owned path. Success criteria: the TUI runtime loop is simpler and more intentionally async-native than before, the remaining bridge is removed or minimized with explicit reasoning, and the resulting behavior still passes imp-tui / imp-cli verification.
