The config system, resource discovery, and hook system all exist as independent modules but aren't connected at agent startup. The TUI's `spawn_agent_for_prompt` assembles a system prompt with empty agents/skills/facts and uses hardcoded thresholds.

## What to implement

1. **Create `AgentBuilder`** in a new file `imp-core/src/builder.rs`:
   - Takes `Config`, `cwd`, `Model`, `api_key`, optional `Role`, optional `TaskContext`
   - Calls `Config::resolve()` to load user/project/env config
   - Applies `config.max_turns`, `config.thinking` to the agent
   - Calls `agent.hooks.load_from_config(config.hooks)` to wire TOML hooks
   - Calls `discover_agents_md()`, `discover_skills()`, `discover_prompts()`
   - Assembles system prompt via `system_prompt::assemble()` with all discovered resources
   - Registers all native tools
   - Returns `(Agent, AgentHandle)`

2. **Wire context thresholds from config** in `agent.rs`:
   - The agent loop currently hardcodes `0.6` and `0.8` for observation masking and compaction thresholds, and `10` for mask window
   - Add `context_config: ContextConfig` field to `Agent`
   - Read thresholds from `self.context_config` instead of hardcoded values
   - `AgentBuilder` sets this from `config.context`

3. **Update imp-tui `app.rs`** to use `AgentBuilder` instead of manual setup in `spawn_agent_for_prompt`

4. **Update imp-cli `main.rs`** to use `AgentBuilder` for print mode and headless mode

## Files
- NEW: `imp-core/src/builder.rs`
- `imp-core/src/lib.rs` (add `pub mod builder`)
- `imp-core/src/agent.rs` (add `context_config` field, use it in the loop)
- `imp-tui/src/app.rs` (use AgentBuilder)
- `imp-cli/src/main.rs` (use AgentBuilder)

## Acceptance criteria
- `AgentBuilder::build()` produces a fully wired agent
- Context thresholds come from config, not hardcoded
- AGENTS.md files and skills included in system prompt
- Hooks from config.toml loaded into HookRunner
- All existing tests pass + new tests for AgentBuilder
