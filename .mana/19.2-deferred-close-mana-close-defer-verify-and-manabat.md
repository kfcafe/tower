## What to implement

When mana close is called with deferred verification (via --defer-verify flag or MANA_BATCH_VERIFY=1 env var), skip the verify step and mark the unit as AwaitingVerify instead of Closed.

### 1. Update CloseOpts (mana/crates/mana-core/src/ops/close.rs)

```rust
pub struct CloseOpts {
    pub reason: Option<String>,
    pub force: bool,
    pub defer_verify: bool,  // NEW
}
```

### 2. Update close() lifecycle

In close(), after pre-close hook (step 1), before verify (step 2):

```rust
if opts.defer_verify {
    unit.status = Status::AwaitingVerify;
    unit.updated_at = Utc::now();
    unit.to_file(&bean_path)?;
    rebuild_index(mana_dir)?;
    return Ok(CloseOutcome::DeferredVerify { unit_id: id.to_string() });
}
```

### 3. Add CloseOutcome::DeferredVerify variant

```rust
DeferredVerify { unit_id: String },
```

### 4. CLI integration (mana/crates/mana-cli/src/commands/close/)

Add --defer-verify flag. Also check MANA_BATCH_VERIFY=1 env var — if set, auto-defer.

### 5. Tests (prefix close_defer)

- close_defer_skips_verify: defer_verify: true → status AwaitingVerify, verify NOT run
- close_defer_returns_outcome: outcome is DeferredVerify  
- close_defer_normal_unchanged: without defer → verify runs as today
