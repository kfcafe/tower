## Problem

`mana doctor` (in `mana/crates/mana-cli/src/commands/doctor.rs`) already checks for stale index, mixed formats, duplicate IDs, orphaned deps, missing parents, stale index entries, cycles, and config findings. However it does NOT check for the converse of stale index entries: **unit files that exist on disk but are missing from the index**. This is the exact desync observed in the bug report.

Also, `doctor --fix` rebuilds the index when stale but doesn't explicitly fix the "file exists, index entry missing" case — though rebuilding would implicitly fix it, the diagnosis is never reported.

## Solution

### 1. Add a "missing from index" check in `collect_issues()`

In `mana/crates/mana-cli/src/commands/doctor.rs`, after Check 7 (StaleIndexEntry, line ~275), add Check 7b:

```rust
// Check 7b: Unit files that exist on disk but aren't in the index
let indexed_ids: HashSet<String> = index.units.iter().map(|e| e.id.clone()).collect();
for id in &existing_ids {
    if !indexed_ids.contains(id) {
        issues.push(Issue::MissingIndexEntry { id: id.clone() });
    }
}
```

### 2. Add the Issue variant

Add to the `Issue` enum:
```rust
MissingIndexEntry { id: String },
```

### 3. Add display/fix handling

In `cmd_doctor()`, add the display case for `MissingIndexEntry`:
```rust
Issue::MissingIndexEntry { id } => {
    println!("  ⚠ Unit file exists for '{id}' but missing from index");
}
```

If `fix` is true and there are any `MissingIndexEntry` issues, the existing index rebuild at the top of `collect_issues` (when stale) should handle it. But if the index isn't detected as stale (timestamp race), force a rebuild when MissingIndexEntry issues are found:

In the fix path, after collecting issues, if any MissingIndexEntry exists:
```rust
if fix && issues.iter().any(|i| matches!(i, Issue::MissingIndexEntry { .. })) {
    let index = Index::build(mana_dir)?;
    index.save(mana_dir)?;
    println!("  ✓ Rebuilt index to include missing entries");
}
```

### 4. Also improve `get_existing_unit_files()` if needed

Check that this function returns unit IDs (not filenames). Looking at the existing code, it should already return IDs since it's used in the StaleIndexEntry check.
