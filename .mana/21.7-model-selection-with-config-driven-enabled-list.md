The model selector currently shows ALL registered models. This is noisy — most users only want 2-4 models. Pi has an `enabledModels` setting that filters the list.

1. Add `enabled_models: Option<Vec<String>>` to Config
2. When opening model selector, filter to only enabled models (if set)
3. Ctrl+L opens model selector (already works)
4. Selecting a model switches immediately for the next message
5. The selected model persists in the session (not just in memory)

Config example:
```toml
enabled_models = ["claude-sonnet-4-6", "claude-opus-4-6", "claude-haiku-4-5"]
```

Also add to settings panel from 21.6.

- imp/crates/imp-core/src/config.rs — add enabled_models field
- imp/crates/imp-tui/src/app.rs — filter model list in open_model_selector
- imp/crates/imp-tui/src/views/model_selector.rs — no changes needed (already works with filtered list)
