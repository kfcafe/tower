Add an optional fast verify step that runs before the full verify gate.

DispatchUnit gains:
- verify_fast: Option<String> — quick check (e.g. cargo check, eslint --fix)

The pool can run verify_fast between agent cycles or after agent completion as a pre-filter before running the expensive verify command. If verify_fast fails, skip the full verify and report the fast failure.

For Rust projects: cargo check catches type errors in seconds. cargo test takes minutes. Don't burn time on full verify when a 5-second check would catch the issue.

This also enables the shift-feedback-left pattern from Stripe Minions: run the cheapest checks first, only escalate to expensive checks when cheap ones pass.

The verify_fast field needs to be read from the mana unit frontmatter. This may require a mana-core change to add verify_fast to the Unit type.
