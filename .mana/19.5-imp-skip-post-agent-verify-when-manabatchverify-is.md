## What to implement

When MANA_BATCH_VERIFY=1 is set, imp skips its post-agent verify and updates the prompt to tell agents not to run the full verify command.

### 1. Skip post-agent verify (imp/crates/imp-cli/src/main.rs)

In run_headless_mode(), change the verify section at the end:

```rust
// Current:
if let Some(verify) = unit.verify.as_deref().map(str::trim).filter(|v| !v.is_empty()) {
    return run_verify_command(verify, &unit.workspace_root).await;
}

// New:
let batch_verify = std::env::var("MANA_BATCH_VERIFY").is_ok();
if !batch_verify {
    if let Some(verify) = unit.verify.as_deref().map(str::trim).filter(|v| !v.is_empty()) {
        return run_verify_command(verify, &unit.workspace_root).await;
    }
}
```

### 2. Update prompt for batch mode (mana/crates/mana-core/src/prompt.rs)

In format_verify_gate(), when MANA_BATCH_VERIFY is set:

```rust
let batch_mode = std::env::var("MANA_BATCH_VERIFY").is_ok();

if batch_mode {
    // Tell agent: verify is handled by orchestrator, use scoped checks, 
    // signal completion with mana close
} else {
    // Existing behavior
}
```

The batch prompt should say:
- "Your verify command is: <cmd>"
- "Batch verify mode: the orchestrator runs this after you exit"
- "Use scoped checks (cargo check -p <crate>) for feedback during work"
- "Signal completion with: mana close <id>"

## Context

imp run_headless_mode flow:
1. Load unit, set up agent with tools
2. Run agent loop (agent works, calls mana close)
3. After agent exits, run verify command → exit code determines success
4. mana runner checks exit code

Step 3 is what we skip. With batch verify, the agent calls mana close (which defers verify per unit 19.2), then imp exits 0, then the mana runner runs batch verify (unit 19.4).
