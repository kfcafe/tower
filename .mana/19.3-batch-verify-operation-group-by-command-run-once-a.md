## What to implement

New operation in mana-core that takes units in AwaitingVerify status, groups them by verify command, runs each unique command once, and returns per-unit pass/fail.

### 1. New file: mana/crates/mana-core/src/ops/batch_verify.rs

```rust
pub struct BatchVerifyResult {
    pub passed: Vec<String>,
    pub failed: Vec<BatchVerifyFailure>,
    pub commands_run: usize,
}

pub struct BatchVerifyFailure {
    pub unit_id: String,
    pub verify_command: String,
    pub exit_code: Option<i32>,
    pub output: String,
    pub timed_out: bool,
}

/// Run batch verification for all AwaitingVerify units.
pub fn batch_verify(mana_dir: &Path) -> Result<BatchVerifyResult>

/// Run batch verification for specific unit IDs.
pub fn batch_verify_ids(mana_dir: &Path, ids: &[String]) -> Result<BatchVerifyResult>
```

### 2. Core logic

1. Load all units in AwaitingVerify status (or given IDs)
2. Group by unit.verify command string (exact match)
3. For each unique command, run once via verify::run_verify_command()
4. Pass → call close() with force: true for full lifecycle (archive, cascade, hooks)
5. Fail → set status back to Open, record failure in unit history

### 3. Register in ops/mod.rs

Add `pub mod batch_verify;`

### 4. Tests (prefix batch_verify)

- batch_verify_groups_by_command: 3 units, 2 share verify → 2 commands run
- batch_verify_passes_close_units: pass → units closed
- batch_verify_fails_reopen_units: fail → units back to Open
- batch_verify_empty_noop: no AwaitingVerify → empty result
- batch_verify_mixed_results: some pass, some fail → correct split
