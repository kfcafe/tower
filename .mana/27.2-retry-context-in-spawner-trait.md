Give the Spawner information about previous attempts so it can differentiate first attempt vs retry.

Add to DispatchUnit or a new SpawnContext:
- attempt_number: u32
- previous_failure: Option<String>  
- previous_notes: Vec<String>

The pool reads attempt history from mana-core before dispatching. First attempt gets exploration-oriented dispatch. Retries get focused dispatch with failure context. This is the Anthropic initializer/coding-agent split as data, not separate agent types.

Requires mana-pool to depend on mana-core for reading attempt history (it already does).
