## What to implement

Wire AgentMode into the agent, builder, and tool execution pipeline. Two layers of enforcement:
1. Registration-time: Don't register disallowed tools
2. Execution-time: Guard in execute_one_tool rejects disallowed tool calls

### 1. Add mode field to Agent struct
In agent.rs, add pub mode: AgentMode to Agent. Default to AgentMode::Full.

### 2. Wire mode in AgentBuilder
In builder.rs build(): read config.mode, set agent.mode. Modify register_native_tools to accept mode and filter. Add retain() method to ToolRegistry.

### 3. Execution-time guard in execute_one_tool
In agent.rs execute_one_tool(), after hook check (L476), before arg validation (L487):
Return error if !self.mode.allows_tool(tool_name) with message like: Tool 'write' is not available in orchestrator mode

### 4. Mode-aware system prompt
In system_prompt.rs identity_layer: use mode to filter tool definitions. Add definitions_for_mode(mode) to ToolRegistry. Append mode instructions.

### Files
- imp/crates/imp-core/src/agent.rs - mode field + execution guard
- imp/crates/imp-core/src/builder.rs - wire mode, filter registration
- imp/crates/imp-core/src/tools/mod.rs - retain() + definitions_for_mode()
- imp/crates/imp-core/src/system_prompt.rs - mode-aware identity layer
