## Task
Add edge case tests for compaction and context management. Currently compaction has 3 tests and context has 7. The verify gate requires at least 16 total.

## compaction tests to add

### compact_empty_messages
Call compact with an empty message slice → should return a summary like "No conversation to summarize" or handle gracefully (not panic).

### compact_single_turn
Only one assistant turn → keep_recent_turns means nothing gets summarized. Should return a summary of just that turn.

### compact_all_recent
If keep_recent_turns >= total turns → no old messages to summarize. Should short-circuit.

### compact_preserves_user_request
Verify the compaction prompt includes the original user request verbatim (this is critical per the spec — failed approaches must be preserved).

## context tests to add

### mask_observations_replaces_content_with_placeholder
After masking, old tool results should contain "[Output omitted — ran {tool}({args_summary}), returned {size}]" format. Verify the exact placeholder format.

### mask_observations_preserves_all_assistant_reasoning
After masking, ALL assistant text blocks must be intact. Only tool result content is replaced.

### context_usage_with_zero_messages
Empty messages → usage.used = 0, ratio = 0.0

### context_usage_near_limit
Messages that bring usage to 99% → ratio ≈ 0.99

### mask_then_compact_flow
Simulate the two-stage flow: first mask at 60%, check result, then compact at 80%. Verify both stages work in sequence without data corruption.

### estimate_tokens_empty_string
estimate_tokens("") → 0

## Implementation notes

- Context tests are pure functions — no mocking needed.
