Create a settings overlay (opened via `/settings` or a hotkey) that shows and allows editing of:

1. **Default model** — dropdown from registered models
2. **Thinking level** — cycle through Off/Low/Medium/High/XHigh
3. **Enabled models** — checkboxes for which models appear in the model selector
4. **Max turns** — number input
5. **Context thresholds** — observation_mask, compaction
6. **Shell backend** — sh/rush

The settings panel should:
- Read current values from Config
- Allow in-session changes (immediate effect)
- Persist changes to user config.toml on save
- Show current values clearly

Implement as a new view in imp-tui/src/views/settings.rs. Follow the pattern of ModelSelectorView — a state struct + Widget impl.

- imp/crates/imp-tui/src/views/settings.rs — new file
