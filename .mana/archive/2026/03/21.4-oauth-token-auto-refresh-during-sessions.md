Currently the TUI does a one-shot `auth_store.resolve()` when spawning an agent. If the OAuth token expires mid-session (common with long conversations), API calls fail with 401.

Fix: Wire token refresh into the provider layer so it happens transparently.

Options (pick the simplest):
A. Pass an `Arc<Mutex<AuthStore>>` to the Anthropic provider so it can call `resolve_or_refresh` before each request
B. Add a pre-request hook in the provider that checks expiry and refreshes
C. Catch 401 errors in the retry loop and trigger a refresh

The infrastructure exists:
- `AuthStore::resolve_or_refresh()` in auth.rs
- `AnthropicOAuth::refresh_token()` in oauth/anthropic.rs
- The Anthropic provider in providers/anthropic.rs makes requests

Files:
- imp/crates/imp-llm/src/providers/anthropic.rs — add token refresh before/during requests
- imp/crates/imp-llm/src/auth.rs — may need to make AuthStore sharable (Arc<Mutex>)
- imp/crates/imp-tui/src/app.rs — pass auth store to provider
- imp/crates/imp-core/src/agent.rs — may need auth store reference
