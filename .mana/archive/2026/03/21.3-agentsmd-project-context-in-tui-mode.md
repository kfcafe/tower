The headless mode builds a rich system prompt via `system_prompt.rs` that includes AGENTS.md, project context, tool descriptions, etc. The TUI mode skips this — `AgentBuilder` has system prompt assembly but `spawn_agent_for_prompt` in app.rs doesn't ensure AGENTS.md is loaded.

Fix:
1. In `spawn_agent_for_prompt` (or App::new), discover and load AGENTS.md files from cwd and parent directories
2. Use `imp_core::system_prompt::build_system_prompt()` (or equivalent) to assemble the full prompt
3. Include the current date, cwd, and project context just like headless mode does

The system prompt should include:
- Base coding assistant instructions
- AGENTS.md content (project root + cwd)
- Current date and working directory
- Available tools list

Files:
- imp/crates/imp-tui/src/app.rs — wire system prompt building into spawn_agent_for_prompt
- imp/crates/imp-core/src/system_prompt.rs — verify/expose prompt building for TUI use
- imp/crates/imp-cli/src/main.rs — may need to pass AGENTS.md content through
