## What to implement
Audit the existing `mana` and `imp` config surfaces against the Tower config contract. Tighten docs, identify gaps where ordinary behavior is still hardcoded, and either fix small issues or create follow-up units for larger ones.

## Files
- `mana/README.md`
- `mana/crates/mana-core/src/config.rs`
- `imp/README.md`
- `imp/imp_core_plan.md`
- `imp/crates/imp-core/src/config.rs`
- root `.mana/` if you need follow-up units for gaps discovered during the audit

## Context
`mana` already uses `.mana/config.yaml` and `imp` already uses user + project TOML config. The goal is not to invent a new config system, but to check whether the current behavior matches Tower policy:
- shared behavior in repo config
- personal defaults in user config
- secrets out of committed config
- env vars and CLI flags layered on top
- ordinary policy changes should not require code edits

## Acceptance criteria
- `mana` and `imp` docs clearly state their config surfaces and override model
- any remaining hardcoded-policy gaps are either fixed or captured as follow-up units
- no secrets are documented as belonging in committed config
- code still builds for the touched crates
