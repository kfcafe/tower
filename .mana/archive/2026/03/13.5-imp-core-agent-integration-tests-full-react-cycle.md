## Task
Add integration tests for the agent loop that use the MockProvider with REAL tool implementations (not mock tools). This tests the full ReAct cycle: LLM returns tool calls → real tools execute against a temp filesystem → results flow back to LLM → final response.

Currently agent tests use a simple EchoTool mock. These integration tests verify real tool behavior end-to-end.

## Tests to write (in agent.rs, under `mod integration` inside `mod tests`)

### 1. agent_reads_and_writes_file
MockProvider returns:
- Turn 0: ToolCall for `write` with `{path: "test.txt", content: "hello world"}`
- Turn 1: ToolCall for `read` with `{path: "test.txt"}`
- Turn 2: TextDelta "The file contains: hello world"

Verify:
- File test.txt is created in temp dir
- Read tool returns "hello world"
- Agent completes successfully with 3 turns

### 2. agent_edit_tool_modifies_file
MockProvider returns:
- Turn 0: ToolCall for `write` with `{path: "src/main.rs", content: "fn main() {\n    println!(\"old\");\n}"}`
- Turn 1: ToolCall for `edit` with `{path: "src/main.rs", oldText: "old", newText: "new"}`
- Turn 2: ToolCall for `read` with `{path: "src/main.rs"}`
- Turn 3: TextDelta "Done"

Verify:
- File content after edit contains "new" not "old"
- Edit tool result includes a diff

### 3. agent_grep_finds_pattern
MockProvider returns:
- Turn 0: ToolCall for `write` with a file containing a known pattern
- Turn 1: ToolCall for `grep` with `{pattern: "the_pattern", path: "."}`
- Turn 2: TextDelta with result

Verify:
- Grep tool result contains the file path and matching line
- Agent completes

### 4. agent_bash_runs_command
MockProvider returns:
- Turn 0: ToolCall for `bash` with `{command: "echo hello && echo world"}`
- Turn 1: TextDelta "Done"

Verify:
- Bash result contains "hello" and "world"
- Details include exit_code: 0

### 5. agent_handles_tool_error_gracefully
MockProvider returns:
- Turn 0: ToolCall for `read` with `{path: "nonexistent.txt"}`
- Turn 1: TextDelta "File not found, let me try something else"

Verify:
- Read tool result has is_error=true
- Agent continues to turn 1 (self-corrects)
- Agent completes successfully
