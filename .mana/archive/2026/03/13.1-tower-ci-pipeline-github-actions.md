## Task
Create a GitHub Actions CI pipeline for the Tower monorepo at `.github/workflows/ci.yml`.

## Requirements
The pipeline runs on push to main and on PRs. It must cover the full Rust workspace.

### Jobs

**check** — `cargo check --workspace`
- Catches compilation errors fast

**test** — `cargo test -p mana-core -p mana-cli -p imp-llm -p imp-core -p imp-lua -p imp-cli`
- Tests all crates that have tests (skip imp-tui — it has 0 tests and TUI tests need special setup)
- Note: imp-lua requires the `mlua` crate with `vendored` feature, which needs a C compiler. The ubuntu-latest runner has gcc.

**clippy** — `cargo clippy --workspace -- -D warnings`
- Fail on warnings

**format** — `cargo fmt --check`
- Fail if unformatted

### Setup per job
```yaml
steps:
  - uses: actions/checkout@v4
  - uses: dtolnay/rust-toolchain@stable
    with:
      components: clippy, rustfmt
  - uses: Swatinem/rust-cache@v2
  - run: <command>
```

### Caveats
- The workspace includes wizard crates (wizard-proto, wizard-store, etc.) which are mostly stubs. Include them in check but they will have 0 tests.
- mlua with vendored feature compiles Lua from source — this is slow (~30s) but reliable.
- Do NOT add MSRV check yet — we have not established one for tower.
- Do NOT add release/publish jobs — not needed yet.

## Reference
Mana already has CI at `mana/.github/workflows/ci.yml` — use it as a template but adapt for the workspace scope.
