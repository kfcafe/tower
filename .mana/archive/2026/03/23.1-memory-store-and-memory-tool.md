Implement the MemoryStore struct and MemoryTool for agent-curated persistent memory.

## What to build

### 1. MemoryStore (`imp-core/src/memory.rs`)

A persistent store backed by a markdown file. Entries separated by `\n§\n`.

```rust
pub struct MemoryStore {
    path: PathBuf,
    entries: Vec<String>,
    char_limit: usize,
}
```

Methods:
- `load(path, char_limit)` — Parse the file, splitting on `§` separator. Create file if missing.
- `save()` — Write entries back, joined by `\n§\n`.
- `add(content)` — Append entry. Error if would exceed char_limit. Reject exact duplicates.
- `replace(old_text, content)` — Find the unique entry containing `old_text` substring (case-sensitive), replace with `content`. Error if 0 or 2+ matches.
- `remove(old_text)` — Find unique entry containing `old_text`, remove it. Error if 0 or 2+ matches.
- `entries()` — Return slice of entries.
- `usage()` — Return `(used_chars, limit)` where used_chars is the total character count of all entries plus separators.
- `render(label)` — Render for system prompt injection with header showing usage percentage.

Format for `render()`:
```
══════════════════════════════════════════════
MEMORY (your personal notes) [67% — 1,474/2,200 chars]
══════════════════════════════════════════════
Entry one text here
§
Entry two text here
```

Security: Before persisting via add/replace, scan content for obvious prompt injection patterns. Block entries containing:
- Lines starting with `<system>`, `[INST]`, `<<SYS>>` or similar injection markers
- Invisible Unicode characters (zero-width spaces, RTL overrides, etc.)

Return a clear error message when blocked.

### 2. MemoryTool (`imp-core/src/tools/memory.rs`)

Implement the `Tool` trait. Parameters:
- `action`: enum "add" | "replace" | "remove" (required)
- `target`: enum "memory" | "user" (required)  
- `content`: string (required for add/replace)
- `old_text`: string (required for replace/remove)

The tool needs to know where the memory files live. Resolve paths:
- `memory` target → `~/.config/imp/memory.md`
- `user` target → `~/.config/imp/user.md`

Use `Config::user_config_dir()` for the base path.

The tool should return JSON in the tool result showing:
```json
{
  "success": true,
  "message": "Added entry to memory",
  "entries": ["entry 1", "entry 2"],
  "usage": "1,474/2,200"
}
```

On error:
```json
{
  "success": false,
  "error": "Memory at 2,100/2,200 chars. Adding this entry (250 chars) would exceed the limit.",
  "entries": ["existing entry 1", "..."],
  "usage": "2,100/2,200"
}
```

### 3. Wire into mod.rs

Add `pub mod memory;` to `imp-core/src/tools/mod.rs` and `pub mod memory;` to `imp-core/src/lib.rs`.

### 4. Tests

Test the MemoryStore in `memory.rs` with `#[cfg(test)]`:
- Load empty file, add entries, verify save/reload round-trip
- Add until at capacity, verify error on overflow
- Replace with unique substring match
- Replace with ambiguous substring → error
- Remove entry
- Duplicate detection
- Security scanning blocks injection patterns
- `render()` output format with usage percentage

Test the MemoryTool (at minimum verify parameter validation and that it delegates to MemoryStore correctly).
