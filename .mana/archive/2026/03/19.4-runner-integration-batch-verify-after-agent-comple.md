## What to implement

Wire batch verify into the mana runner so agents defer verification and the runner handles it centrally.

### 1. Add batch_verify to RunConfig

In mana/crates/mana-cli/src/commands/run/ready_queue.rs (or mod.rs), add to RunConfig:

```rust
struct RunConfig {
    // ... existing fields ...
    batch_verify: bool,  // NEW
}
```

Populate from Config::batch_verify.

### 2. Signal agents to defer verify

In run_single_direct(), when batch_verify is enabled, set env var on spawned agent process:

```rust
if cfg.batch_verify {
    cmd.env("MANA_BATCH_VERIFY", "1");
}
```

This is added in build_direct_command() or run_single_direct() where the Command is built.

### 3. Run batch verify after dispatch

In run_ready_queue_direct(), after the main dispatch loop ends, if batch_verify is enabled:

```rust
if cfg.batch_verify {
    let batch_result = mana_core::ops::batch_verify::batch_verify(mana_dir)?;
    // Apply results to success/failure counts
    // Print results
}
```

### 4. Human-readable output

```
Batch verify: 2 commands, 3/4 units passed
  ✓ cargo build (units: 1, 2, 3)
  ✗ cargo test -p auth (unit: 4) — exit code 1
```

### 5. JSON stream event

```rust
StreamEvent::BatchVerify {
    commands_run: usize,
    passed: Vec<String>,
    failed: Vec<String>,
}
```

## Context

RunConfig struct currently has: max_jobs, timeout_minutes, idle_timeout_minutes, json_stream, file_locking, run_model.

Agent processes are spawned via build_direct_command() returning std::process::Command.

run_ready_queue_direct() is the main dispatch loop. It returns (Vec<AgentResult>, bool). Batch verify should run after this loop completes, in run_once() which calls it.

Note: With batch_verify on, agent exit 0 means "work complete" not "verify passed". The batch_verify results determine actual pass/fail.
