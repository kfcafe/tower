## What to implement

The ManaTool currently accepts an action parameter (status, list, show, create, close, update) with no restrictions. Add mode-aware filtering so that restricted modes can only use allowed mana sub-actions.

### Implementation

In tools/mana.rs, at the top of execute(), after extracting the action string:

1. Get the agent's mode from ToolContext (which needs a mode field added in 22.2, or read from env)
2. Check AgentMode::allows_mana_action(action)
3. If not allowed, return an error: Mana action 'create' is not available in worker mode

Since ToolContext may not have mode yet, the simplest approach: add pub mode: AgentMode to ToolContext (added in 22.2). Then ManaTool checks ctx.mode.allows_mana_action(action).

Alternatively, if 22.2 doesn't add mode to ToolContext, use the IMP_MODE env var as fallback.

### Mana action permissions by mode
- Full: all actions
- Worker: show, update, status, list (can see own unit and log progress, cannot create/close/run)
- Orchestrator: all actions (status, list, show, create, close, update - full coordination)
- Planner: status, list, show, create (can decompose but not close or run)
- Reviewer: no mana access (tool not registered)
- Auditor: status, list, show (read-only inspection)

- imp/crates/imp-core/src/tools/mana.rs - add action filtering at top of execute()
- imp/crates/imp-core/src/tools/mod.rs - add mode to ToolContext if not done in 22.2
