Add agent memory as Layer 6 of the system prompt, and wire it into AgentBuilder.

## What to build

### 1. Add memory parameters to `system_prompt::assemble()`

In `imp-core/src/system_prompt.rs`:

- Add two new parameters to `assemble()` and `assemble_with_mode()`: `memory_block: Option<&str>` and `user_block: Option<&str>`
- Add Layer 6 after Layer 5 (task context)
- Update all existing callers and tests to pass `None, None` for backward compat

### 2. Wire into AgentBuilder

In `imp-core/src/builder.rs`, in `build()`:
1. Load MemoryStore from `~/.config/imp/memory.md` (char limit from config, default 2200)
2. Load MemoryStore from `~/.config/imp/user.md` (char limit from config, default 1400)
3. Call `render()` on each, pass to `assemble_with_mode()`
4. Only load if `config.learning.enabled` is true (default: true)

### 3. Add LearningConfig to Config

In `imp-core/src/config.rs`, add `LearningConfig` struct with:
- `enabled: bool` (default true)
- `skill_nudge_threshold: u32` (default 8)
- `memory_char_limit: usize` (default 2200)
- `user_char_limit: usize` (default 1400)

Add `learning: LearningConfig` to `Config` with `#[serde(default)]`.

### 4. Tests

- Memory blocks appear in assembled prompt after facts/task layers
- Empty memory blocks are skipped
- Layer ordering correct: identity < context < skills < facts < task < memory
- All existing system_prompt tests still pass
