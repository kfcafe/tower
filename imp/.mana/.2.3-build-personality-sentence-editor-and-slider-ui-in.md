  Rewrite as a full TUI implementation prompt. Build a /personality surface inside imp's TUI that makes personality feel like character creation rather than config editing. Center the interaction around the rendered identity sentence, with word-by-word navigation across the editable tokens. The name should open a custom text input; the other editable words should open curated/exhaustive pickers. Add slider controls for the behavioral dimensions defined in the config model, using 5-band semantics and labels that feel understandable in the UI. Include a clear scope toggle so users can switch between editing global personality and project-specific personality. The UI should make it obvious what is currently selected, what scope is active, and what changes are being made. Do not rely on fake example conversations; instead focus on making the sentence, sliders, labels, and picker interactions feel polished and self-explanatory. Add TUI tests where practical and leave the screen well integrated with the rest of imp's command/navigation model.

  2026-04-01T03:56:09.286739+00:00
  Implemented the first working /personality TUI surface. Added a dedicated `views/personality.rs` module with state and rendering for a sentence-centered personality editor. The overlay currently exposes the editable identity slots (name, work style, voice, focus, role), all five slider-backed behavioral dimensions, a scope toggle (global/project), and an apply/save row. Wired it into the app as a new `UiMode::Personality`, added `/personality` to the slash-command help and command palette, and implemented save behavior that writes to either the user config or project `.imp/config.toml` depending on the active scope. Added focused tests for the slash command opening the overlay and for the default rendered sentence. Verified with `cargo check -p imp-tui` and `cargo test -p imp-tui tui_integration_slash_personality_opens_overlay -- --nocapture`. This is intentionally an MVP of the TUI experience: it is a dedicated overlay and supports navigation/edit/save, but it does not yet include richer picker UI, profile management, or extra polish that belongs in later units.
