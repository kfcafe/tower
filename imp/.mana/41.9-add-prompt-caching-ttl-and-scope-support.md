## Task
Enhance prompt caching to support TTL and scope options from the Anthropic API.

## Current state
Imp only supports `{"type": "ephemeral"}` cache control. The API also supports:
- `{"type": "ephemeral", "ttl": "1h"}` — Cache for 1 hour instead of 5 minutes
- `{"type": "ephemeral", "scope": "global"}` — Share cache across users with identical prompts

## Implementation

1. In `crates/imp-llm/src/provider.rs`, extend `CacheOptions`:
```rust
#[derive(Debug, Clone, Default)]
pub struct CacheOptions {
    pub cache_system_prompt: bool,
    pub cache_tools: bool,
    pub cache_recent_turns: usize,
    /// Use 1-hour TTL instead of default 5-minute.
    pub extended_ttl: bool,
    /// Use global scope (shared across users with identical prompts).
    pub global_scope: bool,
}
```

2. In `crates/imp-llm/src/providers/anthropic.rs`, update `CacheControl`:
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
struct CacheControl {
    #[serde(rename = "type")]
    cache_type: String,
    #[serde(skip_serializing_if = "Option::is_none")]
    ttl: Option<String>,
    #[serde(skip_serializing_if = "Option::is_none")]
    scope: Option<String>,
}

fn make_cache_control(options: &CacheOptions) -> Option<CacheControl> {
    Some(CacheControl {
        cache_type: "ephemeral".into(),
        ttl: if options.extended_ttl { Some("1h".into()) } else { None },
        scope: if options.global_scope { Some("global".into()) } else { None },
    })
}
```

3. Update `ephemeral_cache()` and all call sites to use `make_cache_control()` with the options.

4. The `prompt-caching-scope-2026-01-05` beta header (from unit 41.1) is required for scope to work.
