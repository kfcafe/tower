## Task
Add effort level support. The Anthropic API accepts an `effort` parameter in `output_config` that controls how much work the model does. This is separate from thinking tokens.

## Wire format
```json
{
  "output_config": {
    "effort": "medium"
  }
}
```
Valid values: `"low"`, `"medium"`, `"high"`. The beta header `effort-2025-11-24` must be sent.

## Implementation

1. In `crates/imp-llm/src/provider.rs`, add `EffortLevel` enum and add it to `RequestOptions`:
```rust
#[derive(Debug, Clone, Copy, Default, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum EffortLevel {
    Low,
    #[default]
    Medium,
    High,
}

pub struct RequestOptions {
    // ... existing fields ...
    pub effort: Option<EffortLevel>,
}
```

2. In `crates/imp-llm/src/providers/anthropic.rs`:
   - Add `OutputConfig` struct to the request:
```rust
#[derive(Debug, Serialize)]
struct ApiOutputConfig {
    #[serde(skip_serializing_if = "Option::is_none")]
    effort: Option<String>,
}
```
   - Add `output_config` field to `ApiRequest` (skip_serializing_if Option::is_none)
   - In `build_request()`, if `options.effort` is Some, populate it
   - Add `effort-2025-11-24` to `beta_headers()` when effort is set

3. Default in `RequestOptions::default()` should be `effort: None` (no effort param = API defaults to high)

## Tests
- `test_effort_level_serialization` — verify "low"/"medium"/"high" in output_config
- `test_effort_none_omits_field` — verify output_config is absent when effort is None
- `test_effort_adds_beta_header` — verify effort beta header is present when effort is set

## Files
- `crates/imp-llm/src/provider.rs` (modify — add EffortLevel enum and field)
