## Task
Add beta header support to Imp's Anthropic provider. Claude Code sends these headers to unlock critical server-side features. Without them, Imp gets degraded model behavior.

## What to add

In `crates/imp-llm/src/providers/anthropic.rs`, add an `anthropic-beta` header to ALL requests with a comma-separated list of applicable betas:

### Headers to add (all models):
- `interleaved-thinking-2025-05-14` — Enables thinking blocks between tool calls (not just at start). Massive quality improvement.
- `prompt-caching-scope-2026-01-05` — Enables global-scope prompt caching

### Headers to add (conditionally):
- `context-1m-2025-08-07` — Only when model has >200k context window (Sonnet 4.6, Opus 4.6)

## Implementation

1. Add a function `fn beta_headers(model: &ModelMeta) -> Vec&lt;&amp;'static str&gt;` that returns the applicable beta headers for a given model
2. In `stream_response()`, accept a betas parameter and add the header: `.header("anthropic-beta", betas.join(","))`
3. For OAuth requests, merge with existing beta string (currently hardcoded)
4. The non-OAuth path currently sends NO betas — that must change

## Tests
- `test_beta_headers_large_context` — Sonnet 4.6 (1M context) gets context-1m header
- `test_beta_headers_standard_context` — Haiku 3.5 (200k) does NOT get context-1m
- `test_beta_headers_always_includes_interleaved` — all models get interleaved-thinking
