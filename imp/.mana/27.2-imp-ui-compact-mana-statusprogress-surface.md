## Current State
imp already has UI abstractions for `set_status`, `set_widget`, and compact runtime surfaces, and the TUI work in unit `30` is intended to render those surfaces cleanly. This unit should build the mana-specific UX on top of that plumbing so background/native mana work is legible without flooding the transcript.

## Task
Add a mana-specific compact status and progress surface in imp.

Implement a small mana UI surface that:
1. shows active mana work in a compact area
2. updates during start, progress, success, and failure transitions
3. keeps the transcript concise while mana work runs in the background
4. reuses the existing UI abstraction rather than inventing new UI primitives

## Files to Modify
- `crates/imp-core/src/ui.rs` — only if a small mana-oriented widget/status helper is needed
- `crates/imp-core/src/tools/mana.rs` — emit appropriate widget/status updates
- `crates/imp-tui/src/app.rs` — render and update the compact mana surface
- `crates/imp-cli/src/main.rs` — wire the same surface for RPC or headless-compatible display if needed

## Preconditions
- Reuse the generic widget/status rendering foundation from unit `30`
- Reuse the native mana execution path from unit `27.1`

## Scope Boundaries
- Do **not** implement generic widget infrastructure here; use `30`
- Do **not** redesign the full chat layout
- Do **not** add broader planning/review UI here

## Edge Cases
- zero active mana widgets should leave the layout clean
- multiple updates for the same mana job should replace or refresh cleanly
- failure transitions should be visible without spamming the transcript
- compact UI should remain readable in both TUI and RPC-driven flows where practical

## How to Verify
Run: `cd /Users/asher/tower && cargo check -p imp-tui && cargo check -p imp-cli`

## Done When
- active mana work appears in a compact status/widget surface
- the UI updates during start, progress, end, and failure transitions
- mana background work is legible without dominating the main transcript
