## Task
Improve OAuth handling: add the OAuth beta header properly via the beta_headers system (from 41.1), and handle 401 responses by clearing cached tokens and retrying.

## Current state
OAuth detection (`sk-ant-oat` prefix) exists and works. The beta header is hardcoded. No token refresh on 401.

## Implementation

### 1. OAuth beta via beta_headers system
In the `beta_headers()` function added by unit 41.1, add OAuth beta conditionally:
```rust
fn beta_headers(model: &ModelMeta, is_oauth: bool) -> Vec<&'static str> {
    let mut betas = vec![
        "interleaved-thinking-2025-05-14",
        "prompt-caching-scope-2026-01-05",
    ];
    if model.context_window > 200_000 {
        betas.push("context-1m-2025-08-07");
    }
    if is_oauth {
        betas.push("oauth-2025-04-20");
    }
    betas
}
```

Remove the hardcoded `anthropic-beta` header from the OAuth branch in `stream_response()`.

### 2. Handle 401 with token cache clear
In the retry loop, when a 401 is received:
- If using OAuth, the token may have expired
- Clear any cached auth and retry once
- This requires the retry loop to be able to re-resolve the API key

Add 401 to the retryable status codes list:
```rust
fn is_retryable_status(status: reqwest::StatusCode) -> bool {
    matches!(status.as_u16(), 401 | 429 | 500 | 502 | 503 | 529)
}
```

But only retry 401 once (not the full retry count). Track `had_401: bool` and skip on second 401.
