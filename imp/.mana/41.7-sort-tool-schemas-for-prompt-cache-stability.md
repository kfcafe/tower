## Task
Sort tool definitions alphabetically by name before sending to the API. This prevents prompt cache busts when tools are registered in different orders (e.g., due to lazy-loaded tool groups, MCP connections, or plugin load order).

## Why this matters
Anthropic's prompt caching hashes the tool schemas as part of the cache key. If tools arrive in a different order between requests, the cache misses even though the content is identical. Claude Code sorts built-in tools as a contiguous prefix and MCP tools separately after.

## Implementation

1. In `crates/imp-llm/src/providers/anthropic.rs`, in `build_tool_defs()`:
```rust
fn build_tool_defs(tools: &[ToolDefinition], cache: &CacheOptions) -> Vec<ApiToolDef> {
    // Sort tools alphabetically for cache stability
    let mut sorted_tools: Vec<_> = tools.iter().collect();
    sorted_tools.sort_by(|a, b| a.name.cmp(&b.name));
    
    let len = sorted_tools.len();
    sorted_tools.iter().enumerate().map(|(i, t)| {
        let cc = if cache.cache_tools && i == len - 1 {
            ephemeral_cache()
        } else {
            None
        };
        ApiToolDef {
            name: t.name.clone(),
            description: t.description.clone(),
            input_schema: t.parameters.clone(),
            cache_control: cc,
        }
    }).collect()
}
```

That's it — just add the sort. The cache breakpoint stays on the last tool (now deterministically the alphabetically-last one).
