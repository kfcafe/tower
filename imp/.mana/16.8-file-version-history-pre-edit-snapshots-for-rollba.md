Before the first edit to any file in a session, store the original content. Enables rollback to pre-session state if the agent makes bad edits. Lightweight safety net.

## What to implement

1. **Add `FileHistory` struct** in `imp-core/src/tools/mod.rs`:
   ```rust
   pub struct FileHistory {
       originals: HashMap<PathBuf, String>,
   }
   impl FileHistory {
       pub fn snapshot_before_edit(&mut self, path: &Path) -> Result<()> {
           // Only store if not already stored for this path (first edit wins)
           if !self.originals.contains_key(path) {
               if path.exists() {
                   let content = std::fs::read_to_string(path)?;
                   self.originals.insert(path.to_path_buf(), content);
               }
           }
           Ok(())
       }
       pub fn original(&self, path: &Path) -> Option<&str> { ... }
       pub fn rollback(&self, path: &Path) -> Result<()> { ... }  // write original back
       pub fn tracked_files(&self) -> Vec<&Path> { ... }
   }
   ```

2. **Add `file_history: Arc<Mutex<FileHistory>>` to Agent** (or pass through ToolContext alongside FileTracker).

3. **Wire into edit/multi_edit/write tools**: before writing, call `file_history.snapshot_before_edit(path)`. For new files (write to path that doesn't exist), don't snapshot.

4. **Expose via slash command** (future): `/rollback <file>` restores original. Not in scope for this unit — just get the tracking working.
