Wire canonical usage persistence into imp-core session handling.

Scope:
- Add imp-core helpers that append a canonical usage custom entry whenever a model request completes successfully and reports usage.
- Persist enough linkage to relate the usage record to the assistant turn and to dedupe forked/copied history later.
- Ensure stored cost is computed and saved at persistence time.
- Keep failures/non-usage cases non-fatal: usage persistence must not break the main agent flow.

Requirements:
- The canonical entry must be written through the session manager/session JSONL path.
- New persistence logic must live in imp-core, not duplicated in each frontend.
- Continue preserving assistant messages themselves; usage entries complement, not replace, conversation history.
- Expose read helpers for later CLI aggregation.

Watch-outs:
- Session branching/forking must not create unusable ids.
- Avoid double-writing usage for the same turn.
- Respect existing no-session/in-memory behavior.

Primary files likely involved:
- crates/imp-core/src/session.rs
- crates/imp-core/src/agent.rs
- crates/imp-core/src/imp_session.rs
- any new imp-core usage/accounting module from 37.1
