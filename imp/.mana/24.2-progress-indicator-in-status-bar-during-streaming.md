Show real-time progress in the status bar while the agent is streaming: elapsed time and tool call count.

## What to implement

When `app.is_streaming == true`, replace the right side of the status bar with streaming progress info.

### Changes to `views/status.rs`

Add fields to `StatusInfo`:
```rust
pub struct StatusInfo {
    // ... existing fields ...
    pub is_streaming: bool,
    pub elapsed_secs: u64,
    pub tools_completed: u32,
    pub tools_started: u32,
    pub tool_errors: u32,
}
```

In the `StatusBar` `render()` method, when `self.info.is_streaming`:
- Replace the right-side content with streaming info
- Format: `"⠋ 3 tools • 1 error • 0:42  model_name"`
- Use the existing SPINNER animation from tick (add `tick: u64` to StatusInfo)
- Elapsed time format: `M:SS` (e.g., "1:23") or just `0:42`
- Show errors in red if > 0
- Keep model name visible at the far right

### Changes to `app.rs`

In `build_status_info()`:
```rust
fn build_status_info(&self) -> StatusInfo {
    // ... existing code ...
    StatusInfo {
        // ... existing fields ...
        is_streaming: self.is_streaming,
        elapsed_secs: self.turn_tracker.elapsed().as_secs(),
        tools_completed: self.turn_tracker.tool_calls_completed,
        tools_started: self.turn_tracker.tool_calls_started,
        tool_errors: self.turn_tracker.tool_errors,
    }
}
```

### Format logic
```
fn format_elapsed(secs: u64) -> String {
    format!("{}:{:02}", secs / 60, secs % 60)
}

fn streaming_status_text(info: &StatusInfo) -> String {
    let tools = format!("{} tools", info.tools_completed);
    let errors = if info.tool_errors > 0 {
        format!(" • {} error{}", info.tool_errors, if info.tool_errors > 1 { "s" } else { "" })
    } else { String::new() };
    let elapsed = format_elapsed(info.elapsed_secs);
    format!("{tools}{errors} • {elapsed}")
}
```

### Visual layout during streaming
```
left:  cwd │ session
right: ⠋ 3 tools • 0:42 │ $0.12 │ 45% │ sonnet
```

If there are errors:
```
right: ⠋ 3 tools • 1 error │ 0:42 │ $0.12 │ 45% │ sonnet
```

## Acceptance criteria
- Status bar shows tool count and elapsed time during streaming
- Errors shown in red when > 0
- Normal status bar restored when streaming ends
- Spinner animates using existing tick mechanism
- `cargo test -p imp-tui` passes
