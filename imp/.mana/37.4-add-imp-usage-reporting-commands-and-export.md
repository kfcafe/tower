Add the initial imp usage reporting command family.

Required command surface:
- imp usage summary
- imp usage daily
- imp usage models
- imp usage sessions
- imp usage export --format json

Required behavior:
- Read canonical usage custom entries first.
- Fall back to legacy assistant-message usage so older sessions contribute when possible.
- Deduplicate forked/copied session history using the schema/linkage established earlier in the epic.
- Report input/output/cache tokens and stored cost totals.
- Support practical filters like --since, --until, --provider, --model, --session, and --json where they fit the command.
- Make the table output human-friendly and make JSON export machine-friendly.

Constraints:
- Keep the reporting implementation in imp-cli with reusable query helpers in imp-core as needed.
- Do not introduce a separate canonical global ledger.
- If performance is merely acceptable via session scan for v1, that is fine; correctness is more important than indexing.

Primary files likely involved:
- crates/imp-cli/src/main.rs
- crates/imp-core/src/session.rs
- any imp-core usage query helpers introduced earlier
