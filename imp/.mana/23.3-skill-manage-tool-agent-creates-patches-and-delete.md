Implement SkillManageTool so the agent can create, patch, and delete its own skills.

## What to build

### 1. SkillManageTool (`imp-core/src/tools/skill_manage.rs`)

Implement the `Tool` trait. Agent-created skills go to `~/.config/imp/skills/agent/{name}/SKILL.md`.

Parameters:
- `action`: "create" | "patch" | "delete" (required)
- `name`: skill name ‚Äî lowercase, hyphens, max 64 chars (required)
- `content`: full SKILL.md content including frontmatter (for create)
- `old_text`: text to find in existing SKILL.md (for patch)
- `new_text`: replacement text (for patch)

Actions:
- `create` ‚Äî Write `~/.config/imp/skills/agent/{name}/SKILL.md`. Error if already exists. Validate: content must have YAML frontmatter with `name` and `description`.
- `patch` ‚Äî Read the existing SKILL.md, find `old_text`, replace with `new_text`. Error if old_text not found or found multiple times.
- `delete` ‚Äî Remove the skill directory. Only works for agent-created skills (inside `agent/` subdir). Error if trying to delete a non-agent skill.

Name validation (agentskills.io spec):
- Lowercase letters, numbers, hyphens only
- Max 64 chars
- No leading/trailing hyphens
- No consecutive hyphens

### 2. Frontmatter validation

For `create`, parse the YAML frontmatter and verify:
- Has `name` field matching the provided name parameter
- Has non-empty `description` field
- Optional: `license`, `compatibility`, `metadata`

Use a simple approach ‚Äî find `---` delimiters, parse the YAML between them.

### 3. Wire into tools/mod.rs

Add `pub mod skill_manage;` to `imp-core/src/tools/mod.rs`.

### 4. Tests

- Create skill with valid content ‚Üí file written correctly
- Create duplicate ‚Üí error
- Patch existing skill ‚Üí content updated
- Patch with missing old_text ‚Üí error
- Delete agent skill ‚Üí directory removed
- Delete non-agent skill ‚Üí error
- Name validation: reject uppercase, spaces, leading hyphens, consecutive hyphens
- Frontmatter validation: reject missing name/description
