Add the learning loop nudges that prompt the agent to use memory and skill tools.

## What to build

### 1. System prompt text additions

In `imp-core/src/system_prompt.rs`, add a "Memory and Learning" section to Layer 1 (identity_layer), after tool descriptions but before role instructions. Only include when learning is enabled.

```
## Memory & Learning

You have persistent memory across sessions. Use the memory tool to save:
- Environment facts (OS, tools, project setup) → target: memory
- User preferences and corrections → target: user
- Lessons learned and tool quirks → target: memory

When you complete a complex task (5+ tool calls, error recovery, or user
correction), consider saving the approach as a skill via skill_manage.

When you load a skill and find it incomplete or wrong, patch it.

Do NOT save: trivial facts, easily re-discovered info, raw data dumps, or
anything already in AGENTS.md.
```

Add a `learning_enabled: bool` parameter to the assemble functions. When false, skip this text.

### 2. Register memory and skill_manage tools in builder

In `imp-core/src/builder.rs`, in `register_native_tools()`:
- Add `MemoryTool` to the tool registration list
- Add `SkillManageTool` to the tool registration list

These tools should be available in all modes except Reviewer and Auditor (which are read-only). Update the `AgentMode::allowed_tool_names()` lists in config.rs to include "memory" and "skill_manage" for Worker, Orchestrator, Planner, and Full modes.

### 3. OnAgentEnd learning check hook (in builder)

In `AgentBuilder::build()`, when `config.learning.enabled` is true, register a programmatic hook on `on_agent_end` that:
1. Counts the total tool calls in the session messages
2. If count >= `config.learning.skill_nudge_threshold` (default 8), returns a HookResult with a learning check message

The hook result should be a modified_content with the nudge text — but since OnAgentEnd can't inject new messages easily, instead: add a helper that the agent loop can call after the final turn. Specifically:

- Add a `pub fn should_nudge_learning(messages: &[Message], threshold: u32) -> bool` function in a new `imp-core/src/learning.rs` module
- The function counts ToolCall content blocks across all assistant messages
- Returns true if count >= threshold

The actual injection point is in the agent loop (`agent.rs`) — after the loop ends with EndTurn, if `should_nudge_learning()` returns true, send one more user message with the nudge and do one more LLM turn. This is a "bonus turn" — it doesn't count against max_turns.

### 4. Tests

- Verify nudge text appears in system prompt when learning enabled
- Verify nudge text absent when learning disabled
- Test `should_nudge_learning()` with various message counts
- Test that memory and skill_manage tools appear in registered tools
