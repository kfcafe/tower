Wire compacted-history semantics into active context assembly and agent/session startup.

Implementation scope:
- Update the paths that prepare messages for model calls so they use compacted active history rather than blindly replaying raw `SessionEntry::Message` history.
- Ensure resumed sessions and continued conversations after compaction feed the model:
  - compaction summary/boundary-derived active context,
  - preserved recent tail,
  - post-compaction turns.
- Update any TUI/session-loading helpers that currently assume raw `get_messages()` is the entire active conversation.
- Keep transcript/tree/export semantics coherent even if the active model-visible history differs from the full raw persisted transcript.
- Validate interaction with current observation masking and context usage reporting.
- Add tests for:
  - post-compaction follow-up turn uses compacted history
  - resumed compacted session uses compacted history
  - fork from a compacted branch preserves the intended active context semantics

Deliverables:
- Agent/session context assembly that respects compaction boundaries.
- Tests proving future turns after compaction are actually using compacted context.

Out of scope:
- Slash command UX work.
- Remote/provider-specific compaction.
- Auto-compaction triggering.

Done when:
- A successful compaction changes future model-visible context, not just stored metadata.
