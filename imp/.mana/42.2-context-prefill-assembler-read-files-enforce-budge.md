## Task

Create `crates/imp-core/src/context_prefill.rs` — the core assembler that reads files from disk, applies budget constraints, and builds a structured `Message::User` for context injection.

## Types to define

- `FileMode` enum: `Full`, `Tail(usize)`, `Range(usize, usize)`
- `FileSpec` struct: `path: PathBuf`, `mode: FileMode`
- `PrefillConfig` struct: `budget_tokens: usize` (default 50_000), `per_file_tokens: usize` (default 10_000)
- `AssembledContext` struct: `messages: Vec<Message>`, `included_files: Vec<PathBuf>`, `warnings: Vec<String>`, `estimated_tokens: usize`

## Functions to implement

### `assemble_context(specs: &[FileSpec], cwd: &Path, config: &PrefillConfig) -> AssembledContext`

1. For each FileSpec, resolve path relative to `cwd`, read from disk
2. Missing files → add warning, skip (don't error)
3. Apply FileMode (Full = whole file, Tail(n) = last n lines, Range(a,b) = lines a..=b, 1-indexed)
4. Token estimate: `content.len() / 4`
5. Per-file over budget → truncate from end, add `[... truncated]` note
6. Total over budget → skip remaining files, add warning
7. Build single `Message::User` with XML structure:
   ```xml
   <context>
   <file path="src/auth.rs">
   ...content...
   </file>
   </context>
   ```
8. Empty specs or all files missing → return empty messages vec

### `detect_file_paths(text: &str) -> Vec<FileSpec>`

Regex scan for file paths with code extensions. Deduplicate. Parse optional mode suffix: `path.rs:tail:50` → Tail(50), `path.rs:10-50` → Range(10,50). Default mode is Full.

Pattern: `[a-zA-Z_./][a-zA-Z0-9_./+-]*\.(rs|ts|tsx|py|go|js|toml|yaml|yml|json|md|sh|sql|zig|c|cpp|h)`

### Wire into lib.rs

Add `pub mod context_prefill;` to `crates/imp-core/src/lib.rs`.
