Add a `TurnTracker` struct that accumulates statistics during an agent turn, enabling progress indicators and post-turn summaries.

## What to implement

Add a new struct `TurnTracker` in `imp-tui/src/app.rs` (or a new `turn_tracker.rs` module):

```rust
pub struct TurnTracker {
    pub started_at: Instant,
    pub tool_calls_started: u32,
    pub tool_calls_completed: u32,
    pub tool_errors: u32,
    pub files_read: Vec<String>,
    pub files_written: Vec<String>,   // edit + write
    pub files_created: Vec<String>,   // write to new file
    pub commands_run: Vec<String>,    // bash commands (truncated)
    pub searches: u32,                // grep/find/probe calls
}

impl TurnTracker {
    pub fn new() -> Self { /* all zeros, started_at = Instant::now() */ }
    pub fn reset(&mut self) { /* reset all, update started_at */ }
    pub fn elapsed(&self) -> Duration { self.started_at.elapsed() }
    pub fn record_tool_start(&mut self, name: &str, args: &serde_json::Value) { /* increment started, classify tool */ }
    pub fn record_tool_end(&mut self, name: &str, is_error: bool) { /* increment completed/errors */ }
}
```

## Files to modify

1. **New file or section in app.rs**: Define `TurnTracker`
2. **app.rs `App` struct**: Add `pub turn_tracker: TurnTracker` field
3. **app.rs `App::new()`**: Initialize `turn_tracker: TurnTracker::new()`
4. **app.rs `handle_agent_event()`**:
   - `AgentEvent::AgentStart` → `self.turn_tracker.reset()`
   - `AgentEvent::ToolExecutionStart { tool_name, args, .. }` → `self.turn_tracker.record_tool_start(&tool_name, &args)`
   - `AgentEvent::ToolExecutionEnd { .. }` → after attaching output, call `self.turn_tracker.record_tool_end(&tool_name, is_error)`

## Tool classification logic in `record_tool_start`:
- `read` → push `args["path"]` to `files_read`
- `edit` | `multi_edit` → push `args["path"]` to `files_written`
- `write` → push `args["path"]` to `files_created` (or `files_written` if exists — just use `files_written` for simplicity)
- `bash` → push truncated `args["command"]` (first 80 chars) to `commands_run`
- `grep` | `find` | `probe_search` | `probe_extract` → increment `searches`
- everything else → just count it

## Dedup
`files_read` and `files_written` should dedup (use a `BTreeSet<String>` or dedup after insert).

## Acceptance criteria
- `TurnTracker` struct exists with all fields
- `reset()` clears all stats and updates `started_at`
- `record_tool_start` and `record_tool_end` correctly classify tools
- Tracker is wired into all three agent events
- `cargo test -p imp-tui` passes
- Add at least 2 unit tests for tool classification
