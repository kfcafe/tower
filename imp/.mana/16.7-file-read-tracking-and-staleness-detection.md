Track every file read per session. Warn (don't block) on edits to files that haven't been read, and detect external modifications since last read.

## What to implement

1. **Add `FileTracker` struct** in `imp-core/src/tools/mod.rs`:
   ```rust
   pub struct FileTracker {
       reads: HashMap<PathBuf, std::time::SystemTime>,
   }
   impl FileTracker {
       pub fn record_read(&mut self, path: &Path) { ... }
       pub fn was_read(&self, path: &Path) -> bool { ... }
       pub fn is_stale(&self, path: &Path) -> bool { ... }  // compares mtime vs recorded time
   }
   ```

2. **Add `file_tracker: Arc<Mutex<FileTracker>>` to Agent** (or pass through ToolContext). Needs to be shared across tool executions.

3. **Wire into read tool**: after successfully reading a file, call `file_tracker.record_read(path)`.

4. **Wire into edit/multi_edit/write tools**: before applying changes:
   - If `!file_tracker.was_read(path)`, include warning in result: "Warning: editing {file} without reading it first. Consider reading to verify current content."
   - If `file_tracker.is_stale(path)`, include warning: "Warning: {file} was modified externally since last read. Re-read to verify current content."
   - These are warnings, not blocks — the edit still proceeds.

5. **Pass FileTracker via ToolContext** — add an optional `file_tracker` field to ToolContext. Tools that need it can access it.
