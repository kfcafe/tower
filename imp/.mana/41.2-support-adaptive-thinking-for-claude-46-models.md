## Task
Add adaptive thinking support. Claude 4.6+ models are trained to decide their own thinking budget. Sending a fixed `budget_tokens` actually degrades quality on these models.

## Current state
Imp only supports `ThinkingLevel` → fixed `budget_tokens`. The API wire format is:
```json
{ "thinking": { "type": "enabled", "budget_tokens": 32000 } }
```

## What to add
For 4.6+ models, send adaptive thinking instead:
```json
{ "thinking": { "type": "adaptive" } }
```

## Implementation

1. In `crates/imp-llm/src/providers/anthropic.rs`, add a new `ApiThinking` variant:
```rust
#[derive(Debug, Serialize)]
#[serde(tag = "type")]
enum ApiThinking {
    #[serde(rename = "enabled")]
    Enabled { budget_tokens: u32 },
    #[serde(rename = "adaptive")]
    Adaptive,
}
```

2. Add a helper `fn model_supports_adaptive(model_id: &str) -> bool` that returns true for models containing `"4-6"` or `"4.6"` in the ID.

3. In `build_request()`, change the thinking logic:
   - If `ThinkingLevel::Off` → `None` (no change)
   - If model supports adaptive AND thinking level is Medium or higher → `Some(ApiThinking::Adaptive)`
   - If model supports adaptive AND thinking level is Low/Minimal → `Some(ApiThinking::Enabled { budget_tokens })` (small budgets still use fixed)
   - Otherwise (older models) → existing `Some(ApiThinking::Enabled { budget_tokens })` behavior

4. When using adaptive thinking, `max_tokens` should be set to the model's `max_output_tokens` (not capped at 8192) since the model manages its own budget.
