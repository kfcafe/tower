When read/edit/multi_edit receive a path that doesn't exist, search for similar filenames and suggest alternatives. Saves a round trip — the model can self-correct from the suggestion.

## What to implement

1. **Add `pub fn suggest_similar_files`** in `imp-core/src/tools/mod.rs`:
   - Takes the missing path and cwd
   - Extracts the filename component
   - Runs `fd` (or walkdir fallback) to find files with similar names in the project
   - Ranks by Levenshtein edit distance (implement a simple Levenshtein function — ~15 lines, don't add a dep)
   - Returns up to 3 suggestions as strings

2. **Wire into read tool** (`tools/read.rs`): when file not found, call suggest_similar_files and include suggestions in the error message:
   ```
   File not found: src/atuh/middleware.rs
   Did you mean:
     - src/auth/middleware.rs
     - src/auth/middleware_test.rs
   ```

3. **Wire into edit and multi_edit tools**: same pattern — on file not found, suggest alternatives.

4. **Levenshtein distance**: implement inline in `tools/mod.rs`. Simple DP matrix, ~15 lines. Don't add a crate for this.

## Key files
- `imp-core/src/tools/mod.rs` — add suggest_similar_files and levenshtein
- `imp-core/src/tools/read.rs` — wire into file-not-found error path

- `imp-core/src/tools/multi_edit.rs` — wire into file-not-found error path
