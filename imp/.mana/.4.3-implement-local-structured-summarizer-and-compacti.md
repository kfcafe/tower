Implement the local structured summarizer and compaction execution path in `imp-core`.

Implementation scope:
- Add the first real local/manual compaction executor in `imp-core`.
- Generate a structured handoff summary for the summarized prefix only.
- Use a bounded schema oriented toward engineering continuity, e.g.:
  - Goal
  - User constraints/preferences
  - Completed work
  - Current state
  - Key decisions
  - Relevant files
  - Errors/warnings
  - Next step
- Ensure summary generation consumes the safe pre-reduced history produced by `.4.2`.
- Persist a real `SessionEntry::Compaction` for manual compaction with useful metadata (`summary`, `first_kept_id`, token counts if available).
- Add overflow recovery: if the summarization request itself is too large, trim older grouped history and retry rather than failing permanently.
- Define clear fallback behavior if summary generation fails after retries.
- Add tests for:
  - successful manual compaction
  - overflow retry path
  - too-little-history behavior
  - multiple compactions on one branch (at least basic correctness)

Deliverables:
- Local compaction executor in `imp-core`.
- Structured summary prompt/format and tests.
- Persisted compaction entries that later jobs can render/use.

Out of scope:
- TUI command integration.
- Provider-specific remote compaction.
- Auto-compaction thresholds.

Done when:
- `imp-core` can perform a real local/manual compaction and persist the result safely.
