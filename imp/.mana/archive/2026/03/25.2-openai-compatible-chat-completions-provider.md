## Task
Create a new `OpenAiCompatProvider` that implements the `Provider` trait using the OpenAI Chat Completions API format (`/v1/chat/completions`). This is NOT the same as the existing `OpenAiProvider` which uses the newer Responses API (`/v1/responses`). Most third-party providers (DeepSeek, Groq, Together, Mistral, xAI, OpenRouter, Fireworks) use the Chat Completions format.

## Files
- `imp/crates/imp-llm/src/providers/openai_compat.rs` (create — new provider implementation)
- `imp/crates/imp-llm/src/providers/mod.rs` (modify — add module, update create_provider)

## Step 1: Create openai_compat.rs

Create `imp/crates/imp-llm/src/providers/openai_compat.rs` with an implementation of the OpenAI Chat Completions streaming API. Key differences from the Responses API used by `openai.rs`:

**Request format** — Chat Completions uses `messages` array:
```json
{
  "model": "deepseek-chat",
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": [{"type": "text", "text": "..."}, {"type": "image_url", "image_url": {"url": "data:..."}}]},
    {"role": "assistant", "content": "...", "tool_calls": [{"id": "...", "type": "function", "function": {"name": "...", "arguments": "..."}}]},
    {"role": "tool", "tool_call_id": "...", "content": "..."}
  ],
  "tools": [{"type": "function", "function": {"name": "...", "description": "...", "parameters": {...}}}],
  "stream": true,
  "stream_options": {"include_usage": true},
  "temperature": 0.7,
  "max_tokens": 4096
}
```

**SSE response format** — each line is `data: {json}`:
```
data: {"id":"chatcmpl-...","choices":[{"delta":{"role":"assistant","content":"Hello"},"index":0}]}
data: {"id":"chatcmpl-...","choices":[{"delta":{"content":" world"},"index":0}]}
data: {"id":"chatcmpl-...","choices":[{"delta":{"tool_calls":[{"index":0,"id":"call_...","type":"function","function":{"name":"read","arguments":""}}]},"index":0}]}
data: {"id":"chatcmpl-...","choices":[{"delta":{"tool_calls":[{"index":0,"function":{"arguments":"{\"path\":"}}]},"index":0}]}
data: {"id":"chatcmpl-...","choices":[{"delta":{},"finish_reason":"stop"}],"usage":{"prompt_tokens":10,"completion_tokens":20,"total_tokens":30}}
data: [DONE]
```

**Structure the file like the existing `anthropic.rs` and `openai.rs`:**
1. Wire-format request types (serde Serialize)
2. Wire-format response types (serde Deserialize)
3. `build_request()` function converting `Context` + `RequestOptions` → wire format
4. `stream_response()` function that does the HTTP POST and parses SSE into `StreamEvent`s
5. `OpenAiCompatProvider` struct and `Provider` impl

**The struct:**
```rust
pub struct OpenAiCompatProvider {
    client: reqwest::Client,
    provider_id: String,
    base_url: String,
    models: Vec<ModelMeta>,
}

impl OpenAiCompatProvider {
    pub fn new(provider_id: &str, base_url: &str, models: Vec<ModelMeta>) -> Self {
        Self {
            client: reqwest::Client::new(),
            provider_id: provider_id.to_string(),
            base_url: base_url.to_string(),
            models,
        }
    }
}
```

**Message conversion:**
- `Message::User` → `{"role": "user", "content": "..."}` (or content array for images)
- `Message::Assistant` with tool calls → `{"role": "assistant", "content": "...", "tool_calls": [...]}`
- `Message::Assistant` without → `{"role": "assistant", "content": "..."}`
- `Message::ToolResult` → `{"role": "tool", "tool_call_id": "...", "content": "..."}`

**Tool call handling:**
- Tool calls arrive incrementally in the `choices[0].delta.tool_calls` array
- Each tool call has an `index` — use a HashMap to accumulate by index
- First chunk has `id` and `function.name`, subsequent chunks append to `function.arguments`
- Emit `ToolCallStart` on first chunk, `ToolCallDelta` on argument chunks

**Thinking/reasoning support:**
- Some providers (DeepSeek) send reasoning in `choices[0].delta.reasoning_content`
- Emit `ThinkingDelta` when this field is present
- This is optional — not all compatible providers support it

**The streaming function:**
```rust
fn stream_response(
    client: reqwest::Client,
    base_url: String,
    model_id: String,
    api_key: String,
    request: ApiRequest,
) -> Pin<Box<dyn Stream<Item = Result<StreamEvent>> + Send>> {
    let (tx, rx) = tokio::sync::mpsc::channel(256);
    tokio::spawn(async move {
        let url = format!("{base_url}/v1/chat/completions");
        let resp = client
            .post(&url)
            .bearer_auth(&api_key)
            .json(&request)
            .send()
            .await;
        // ... parse SSE lines, emit StreamEvents via tx ...
    });
    Box::pin(tokio_stream::wrappers::ReceiverStream::new(rx))
}
```

Use the same SSE parsing pattern as the existing providers: read the response as bytes stream, split on `\n\n`, parse `data:` lines, handle `data: [DONE]`.

**Provider trait impl:**
```rust
#[async_trait]
impl Provider for OpenAiCompatProvider {
    fn stream(&self, model: &Model, context: Context, options: RequestOptions, api_key: &str)
        -> Pin<Box<dyn Stream<Item = Result<StreamEvent>> + Send>> {
        let request = build_request(model, context, options);
        stream_response(self.client.clone(), self.base_url.clone(), model.meta.id.clone(), api_key.to_string(), request)
    }

    async fn resolve_auth(&self, auth: &AuthStore) -> Result<ApiKey> {
        auth.resolve(&self.provider_id)
    }

    fn id(&self) -> &str {
        &self.provider_id
    }

    fn models(&self) -> &[ModelMeta] {
        &self.models
    }
}
```

## Step 2: Update providers/mod.rs

Add the module and update `create_provider`:

```rust
pub mod anthropic;
pub mod google;
pub mod openai;
pub mod openai_compat;

use crate::model::{ApiStyle, ModelMeta, ProviderMeta, ProviderRegistry};
use crate::provider::Provider;

pub use anthropic::AnthropicProvider;
pub use google::GoogleProvider;
pub use openai::OpenAiProvider;
pub use openai_compat::OpenAiCompatProvider;

/// Create a provider by name, using the provider registry for metadata.
pub fn create_provider(name: &str) -> Option<Box<dyn Provider>> {
    let registry = ProviderRegistry::with_builtins();
    let meta = registry.find(name);

    match meta.map(|m| m.api_style) {
        Some(ApiStyle::Anthropic) => Some(Box::new(AnthropicProvider::new())),
        Some(ApiStyle::OpenAi) => Some(Box::new(OpenAiProvider::new())),
        Some(ApiStyle::Google) => Some(Box::new(GoogleProvider::new())),
        Some(ApiStyle::OpenAiCompat) => {
            let base_url = meta.unwrap().api_base_url.unwrap_or("https://api.openai.com");
            let model_registry = crate::model::ModelRegistry::with_builtins();
            let models: Vec<ModelMeta> = model_registry
                .list_by_provider(name)
                .into_iter()
                .cloned()
                .collect();
            Some(Box::new(OpenAiCompatProvider::new(name, base_url, models)))
        }
        None => None,
    }
}
```
