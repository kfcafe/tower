## Current State
imp already has a native `mana` tool and some background-run support in `crates/imp-core/src/tools/mana.rs`, but the agent still falls back to `bash` for mana orchestration more often than it should. The missing piece is a compact, trustworthy in-process orchestration surface: a parent agent should be able to start mana work, inspect run state later in the same session, and avoid shelling out for common mana operations.

## Task
Narrow this unit to the native orchestration core only.

Implement or tighten the native `mana` tool so it can:
1. expose the high-value orchestration actions the agent actually needs in-session
2. start mana runs natively instead of forcing `bash "mana ..."`
3. keep session-local run state so later tool calls can inspect or reference prior native runs
4. support follow-up inspection paths for started work (for example: status, logs, run state, evaluate, or equivalent)
5. stay compact in schema and tests; do not expand into generic UI or parent/child synthesis behavior here

## Files to Modify
- `crates/imp-core/src/tools/mana.rs` — native mana tool actions, run-state plumbing, output contract
- `crates/imp-core/src/tools/mod.rs` — any shared tool state/context wiring needed
- `crates/imp-core/src/agent.rs` — only if the agent loop needs small support for session-local mana run state

## Scope Boundaries
- Do **not** build the compact UI surface here; that belongs in `27.2`
- Do **not** build non-blocking completion delivery here; that belongs in `27.3`
- Do **not** build broader delegated-child-job synthesis here; that belongs in `39` / `40`
- Prefer adding a few high-value orchestration actions over a large schema expansion

## Edge Cases
- Starting a background/native run twice in one session should not corrupt run state
- Inspecting run state when no native run exists should return a clear, compact result
- Tool output should make the native path obviously preferable to `bash` when an equivalent action exists
- Tests should cover both action availability and stored run-state behavior
