Implement the session_search tool so the agent can search past conversations.

## What to build

### 1. SessionSearchTool (`imp-core/src/tools/session_search.rs`)

Implement the `Tool` trait.

```
name = "session_search"
label = "Session Search"
description = "Search past conversations. Use when you need to recall something discussed in a previous session."
readonly = true
```

Parameters:
```json
{
    "type": "object",
    "required": ["query"],
    "properties": {
        "query": {
            "type": "string",
            "description": "Search query (supports AND, OR, NOT, quoted phrases)"
        },
        "limit": {
            "type": "integer",
            "description": "Max results (default: 5)"
        }
    }
}
```

Execute:
1. Open SessionIndex from `~/.local/share/imp/session_index.db`
2. Call `search(query, limit)`
3. Format results as readable text:

```
Found 3 results for "kubernetes deploy":

[1] Session from 2026-03-20 (~/projects/myapp, 45 messages)
    First: "Help me set up k8s deployment"
    ...snippet with <b>highlights</b>...

[2] Session from 2026-03-15 (~/projects/infra, 23 messages)
    ...
```

If no results, return "No past sessions match that query."

If the index database doesn't exist yet, return "No sessions indexed yet. Session search becomes available after your first conversation."

### 2. Wire into tools/mod.rs and builder.rs

- Add `pub mod session_search;` to `imp-core/src/tools/mod.rs`
- Add `SessionSearchTool` to `register_native_tools()` in builder.rs
- Add "session_search" to allowed tools for all modes (it's read-only)

### 3. Tests

- Tool returns formatted results for matching query
- Tool returns "no results" message for non-matching query
- Tool handles missing database gracefully
- Parameter validation (query is required)
