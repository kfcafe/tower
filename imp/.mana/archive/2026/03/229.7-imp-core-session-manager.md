JSONL-based session persistence with tree structure for branching.

## What to implement

### File: `crates/imp-core/src/session.rs`

**SessionEntry enum:**
- Header { version: u32, created_at: u64, cwd: String }
- Message { id: String, parent_id: Option<String>, message: Message }
- Compaction { id: String, parent_id: Option<String>, summary: String, first_kept_id: String, tokens_before: u32, tokens_after: u32 }
- Custom { id: String, parent_id: Option<String>, custom_type: String, data: Value }
- Label { entry_id: String, label: String }

All entries serialize as one JSON object per line. Each line has a `type` field for discriminated deserialization.

**SessionManager struct:**
```rust
pub struct SessionManager {
    entries: Vec<SessionEntry>,
    leaf_id: Option<String>,    // current position in the tree
    path: Option<PathBuf>,      // None for in-memory
}
```

**Methods:**
- `SessionManager::new(cwd, session_dir)` — create new session, generate UUID filename, write header
- `SessionManager::open(path)` — parse JSONL file, build entry list
- `SessionManager::in_memory()` — no persistence
- `SessionManager::continue_recent(cwd, session_dir)` — find most recent session file for this cwd
- `append(&mut self, entry)` — add entry, set parent_id to current leaf_id, update leaf_id, append to file
- `get_branch(&self) -> Vec<&SessionEntry>` — walk parent_ids from leaf to root, reverse to get chronological order
- `get_messages(&self) -> Vec<&Message>` — extract messages from current branch (for LLM context)
- `get_tree(&self) -> Tree` — build full tree structure from all entries
- `navigate(&mut self, target_id)` — change leaf_id to target (for branch switching)
- `fork(&self, entry_id, new_path)` — create new session file with entries up to entry_id
- `list(session_dir) -> Vec<SessionInfo>` — list all session files with metadata (first message, message count, cwd, timestamp)

**SessionInfo struct:** { id, path, first_message: Option<String>, message_count, cwd, created_at, updated_at }

**ID generation:** Use UUID v4 (add `uuid` crate).

**File organization:**
- Session dir: `~/.local/share/imp/sessions/`
- Subdirectories by URL-encoded cwd path (e.g., `%2Fhome%2Fuser%2Fproject/`)
- Each session: `{uuid}.jsonl`

**Tree struct** for get_tree():
```rust
pub struct TreeNode {
    pub entry: SessionEntry,
    pub children: Vec<TreeNode>,
}
```

## Acceptance criteria
- Create session, append messages, save to file, reopen from file — messages match
- Branch test: append 5 messages, navigate to message 3, append 2 more — get_branch returns the correct 5 entries (3 original + 2 new branch)
- Fork test: fork at message 3, new file contains only entries up to message 3
- list() finds sessions and returns correct metadata
- continue_recent() finds the most recently modified session for a given cwd
- In-memory mode works without touching filesystem
- Handles malformed JSONL lines gracefully (skip bad lines, log warning)
