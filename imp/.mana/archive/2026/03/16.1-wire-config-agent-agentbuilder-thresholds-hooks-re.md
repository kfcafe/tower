The config system, resource discovery, and hook system all exist as independent modules but aren't connected at agent startup. The TUI's `spawn_agent_for_prompt` assembles a system prompt with empty agents/skills/facts and uses hardcoded thresholds.

## What to implement

1. **Create `AgentBuilder`** in `imp-core/src/builder.rs`:
   - Takes Config, cwd, Model, api_key, optional Role, optional TaskContext
   - Applies `config.max_turns`, `config.thinking` to the agent
   - Calls `agent.hooks.load_from_config(config.hooks)` to wire TOML hooks
   - Calls `discover_agents_md()`, `discover_skills()`, `discover_prompts()`
   - Assembles system prompt via `system_prompt::assemble()` with all discovered resources
   - Registers all native tools (copy from `App::register_native_tools` in imp-tui)
   - Returns `(Agent, AgentHandle)`

2. **Wire context thresholds from config** in `agent.rs`:
   - Add `pub context_config: ContextConfig` field to `Agent` (default via `ContextConfig::default()`)
   - In the agent loop, replace hardcoded `0.6` with `self.context_config.observation_mask_threshold`
   - Replace hardcoded `0.8` with `self.context_config.compaction_threshold`
   - Replace hardcoded `10` with `self.context_config.mask_window`

3. **Update imp-tui** `app.rs` `spawn_agent_for_prompt` to use AgentBuilder
4. **Update imp-cli** `main.rs` print mode and headless mode to use AgentBuilder

## Key files to read
- `imp-core/src/config.rs` — Config, ContextConfig structs
- `imp-core/src/resources.rs` — discover_agents_md, discover_skills, discover_prompts
- `imp-core/src/system_prompt.rs` — assemble()
- `imp-core/src/agent.rs` — Agent struct, search for 0.6, 0.8, 10
- `imp-tui/src/app.rs` — spawn_agent_for_prompt, register_native_tools
- `imp-cli/src/main.rs` — print mode and headless mode agent creation
