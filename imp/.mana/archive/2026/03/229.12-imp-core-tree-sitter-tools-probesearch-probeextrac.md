Language detection: by file extension (standard mapping).

### probe_search tool
- Params: { query: String, path?: String, language?: String, max_results?: u32, max_tokens?: u32, allow_tests?: bool, exact?: bool }
- Implementation:
  1. Use `grep` crate (or ripgrep) to find lines matching query keywords
  2. For each match, parse the file with tree-sitter
  3. Find the enclosing function/class/struct for each matched line
  4. Deduplicate blocks (same function matched multiple times)
  5. Sort by relevance (number of keyword hits)
  6. Return complete code blocks with file:line headers
- Supports Elasticsearch-style queries: AND, OR, NOT, phrases ("..."), hints (ext:, dir:, lang:)
- Apply truncation to total output

### probe_extract tool
- Params: { targets: String[] } where each target is "file:line", "file#symbol", or "file:start-end"
- For each target:
  1. Parse the file with tree-sitter
  2. Find the AST node at the specified line or matching the symbol name
  3. Walk up the tree to find the enclosing complete block (function, class, struct, impl)
  4. Return the full source text of that block
- Multiple targets in one call

### scan tool
- Params: { action: "extract"|"build"|"scan", files?: String[], directory?: String, task?: String, max_depth?: u32, format?: "markdown"|"json" }
- "extract" single file: list all functions, types, imports with line numbers
- "build" multiple files: extract structure from each, optionally filter by task relevance
- "scan" directory: walk directory, extract structure from all source files
- Output format: markdown (default) or JSON

### ast_grep tool
- Params: { pattern: String, lang?: String, path?: String, replace?: String, dry_run?: bool }
- Pattern syntax: $NAME matches any identifier, $$$ARGS matches any arguments
- Implementation: parse the pattern into a tree-sitter query, search for matches
- If replace provided: apply structural replacement (write mode, not readonly)
- If dry_run: show what would change

## Acceptance criteria
- probe_search: search for "fn main" in a Rust file → returns the complete main function
- probe_extract: extract "file.rs:10" → returns the enclosing function at line 10
- probe_extract: extract "file.rs#my_function" → returns the named function
- scan extract: scan a Rust file → lists all functions, structs, impls with line numbers
- ast_grep: search for "$X.unwrap()" → finds all unwrap calls
- Language detection works for all supported extensions
- Truncation applied to large outputs
