Add SQLite FTS5 session indexing for cross-session search.

## What to build

### 1. Add rusqlite dependency

In `imp/crates/imp-core/Cargo.toml`, add:
```toml
rusqlite = { version = "0.31", features = ["bundled", "fts5"] }
```

IMPORTANT: Check the latest version of rusqlite on crates.io first. Use whatever is current. The `bundled` feature compiles SQLite from source (no system dependency). The `fts5` feature enables full-text search.

### 2. SessionIndex (`imp-core/src/session_index.rs`)

```rust
pub struct SessionIndex {
    db: rusqlite::Connection,
}
```

Schema:
```sql
CREATE TABLE IF NOT EXISTS sessions (
    id TEXT PRIMARY KEY,
    cwd TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    message_count INTEGER NOT NULL,
    first_message TEXT
);

CREATE VIRTUAL TABLE IF NOT EXISTS session_content USING fts5(
    session_id,
    content,
    tokenize='porter unicode61'
);
```

Methods:
- `open(path: &Path) -> Result<Self>` — Open or create the database. Run migrations on open. Path: `~/.local/share/imp/session_index.db`
- `index_session(session: &SessionManager) -> Result<()>` — Extract text from a session and index it. Extract: user message text, assistant text (not tool results — too noisy), compaction summaries. Concatenate into a single content string. Upsert into both tables.
- `search(query: &str, limit: usize) -> Result<Vec<SessionSearchHit>>` — FTS5 search. Use `snippet()` function for highlighted excerpts. Order by rank (relevance), then by created_at desc.
- `is_indexed(session_id: &str) -> bool` — Check if a session is already indexed.

```rust
pub struct SessionSearchHit {
    pub session_id: String,
    pub cwd: String,
    pub created_at: u64,
    pub snippet: String,
    pub message_count: usize,
    pub first_message: Option<String>,
}
```

### 3. Wire into lib.rs

Add `pub mod session_index;` to `imp-core/src/lib.rs`.

### 4. Tests

- Create index, insert session, search finds it
- FTS5 query with AND/OR/NOT
- Search with no results returns empty vec
- Multiple sessions, search ranks by relevance
- Duplicate indexing is idempotent (upsert)
- snippet() returns highlighted excerpts
