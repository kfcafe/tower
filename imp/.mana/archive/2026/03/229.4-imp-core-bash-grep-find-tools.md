Implement the execution and search tools. These depend on the Tool trait from 229.3.

## bash.rs — Shell command execution

Params: { command: String, timeout?: u32 (seconds) }

Implementation:
- Spawn via `tokio::process::Command::new("sh").arg("-c").arg(&command)`
- Set cwd to ToolContext.cwd
- Stream stdout and stderr together (merged, line by line) via `update_tx`
- Respect CancellationToken — on cancel, send SIGTERM to process group, wait 5s, then SIGKILL
- Default timeout: 120 seconds. On timeout, kill process group.
- Process group management: spawn with `pre_exec` to call `setsid()` so killing the group kills all children
- Capture exit code
- Apply truncation (tail mode — for command output, the end matters more)
- Return: stdout+stderr text, exit code, whether it was killed/timeout

Edge cases:
- Command produces no output for a long time (idle) — don't timeout on idle, only on total duration
- Command produces massive output — stream partial results via update_tx, truncate final result
- Binary output — detect and skip (check for null bytes)

## grep.rs — Search file contents via ripgrep

Params: { pattern: String, path?: String, glob?: String, ignore_case?: bool, context?: u32, limit?: u32, literal?: bool }

Implementation:
- Shell out to `rg` (ripgrep) with appropriate flags
- If `rg` not found, fall back to a basic Rust implementation using `grep` crate or line-by-line regex
- Default path: cwd
- Default limit: 100 matches
- Respect .gitignore (rg does this by default)
- Format output: `file:line: content`
- Truncate long lines to 500 chars
- Truncate total output to 50KB / 2000 lines (head mode)
- Return: matching lines with file paths and line numbers

## find.rs — Find files by glob

Params: { pattern: String, path?: String, limit?: u32 }

Implementation:
- Shell out to `fd` with glob pattern
- If `fd` not found, fall back to `walkdir` + `glob` crate
- Default path: cwd
- Default limit: 1000 results
- Respect .gitignore
- Return sorted file paths, one per line
- Truncate to limit
