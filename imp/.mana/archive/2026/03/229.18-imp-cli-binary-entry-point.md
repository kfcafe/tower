The `imp` binary. CLI argument parsing, mode dispatch, startup sequence.

## Crate: `crates/imp-cli/`

### Dependencies
- clap = { version = "4", features = ["derive"] }
- imp-core, imp-llm, imp-lua, imp-tui (paths)
- tokio, tracing, tracing-subscriber

### CLI interface

```
imp [options] [@files...] [messages...]

Modes:
  (default)          Interactive TUI
  -p, --print        Print response and exit
  --mode rpc         JSON-lines over stdin/stdout
  --mode json        Output all events as JSON lines
  imp run <unit-id>  Headless mode (mana unit)
  imp login          OAuth login flow

Options:
  --provider <name>          LLM provider
  --model <pattern>          Model (supports provider/id and :thinking shorthand)
  --thinking <level>         off|minimal|low|medium|high|xhigh
  --api-key <key>            API key override
  -c, --continue             Continue most recent session
  -r, --resume               Browse and select session
  --session <path>           Use specific session
  --no-session               Ephemeral mode
  --tools <list>             Enable specific tools
  --no-tools                 Disable all built-in tools
  --system-prompt <text>     Replace default system prompt
  --verbose                  Verbose startup
  -h, --help                 Show help
  -v, --version              Show version

Package commands:
  imp login [provider]       OAuth login
  imp config                 Edit config
  imp list-models [search]   List available models
```

### Startup sequence

1. Parse CLI args (clap)
2. Initialize tracing/logging
3. Load config (merge hierarchy: CLI → env → project → user → defaults)
4. Initialize AuthStore (load ~/.config/imp/auth.json)
5. Initialize ModelRegistry (built-in + custom models)
6. Resolve model (CLI flag → config → first available)
7. Discover resources (AGENTS.md, skills, prompts)
8. Load Lua extensions
9. Initialize ToolRegistry (built-in + shell + Lua tools)
10. Resolve session (--continue, --resume, --session, or new)
11. Assemble system prompt
12. Dispatch to mode:
    - Interactive → start TUI event loop
    - Print → send prompt, stream response to stdout, exit
    - RPC → start JSON-lines protocol on stdin/stdout
    - Headless → load mana unit, run agent, exit

### Print mode (-p)
- Send the prompt (from args or stdin)
- Stream assistant text to stdout
- Stream tool calls to stderr (or suppress)
- Exit with 0 on success, 1 on error
- No session persistence

### RPC mode (--mode rpc)
- Read JSON commands from stdin (one per line)
- Write JSON events to stdout (one per line)
- Commands: prompt, steer, follow_up, cancel, get_commands, set_model, compact
- Events: all AgentEvent variants + ui_request for interactive prompts
- LF-delimited (not general line separators — important for correct framing)

### Headless mode (imp run <unit-id>)
- Load mana unit by ID from .mana/
- Assemble task context (description, deps, history, facts)
- Run agent with task context in system prompt
- Stream events to stdout as JSON-lines
- Accept commands on stdin (cancel, steer)
- Run verify gate on completion
- Exit 0 (verify passed), 1 (verify failed), 2 (error)

### File arguments
- @file includes file content in the message
- Multiple @files supported
- Image files detected by extension, included as images

## Acceptance criteria
- `cargo build -p imp-cli` produces a working binary
- `imp --help` shows usage
- `imp --version` shows version
- `imp -p "Hello"` sends prompt and prints response (requires API key)
- `imp --list-models` shows available models
- `imp login` starts OAuth flow
- Startup completes in <100ms (no API key required for startup)
