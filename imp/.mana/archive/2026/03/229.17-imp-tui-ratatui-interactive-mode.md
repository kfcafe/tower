Terminal UI built on ratatui + crossterm. The daily driver interface.

## Crate: `crates/imp-tui/`

### Dependencies
- ratatui = "0.29"
- crossterm = "0.28"
- syntect = "5" (syntax highlighting)
- imp-core, imp-llm, imp-lua (paths)
- tokio, serde_json

### Architecture

**Event loop** (in `src/app.rs`):
- crossterm raw mode + alternate screen
- Main loop: poll crossterm events (keyboard, resize) + agent events (from mpsc channel)
- On keyboard event: route to editor or current UI component
- On agent event: update app state, trigger re-render
- Render at ~30fps or on event (whichever is sooner)
- Terminal resize handled by ratatui automatically

**App state:**
```rust
pub struct App {
    messages: Vec<DisplayMessage>,    // rendered message history
    editor: EditorState,              // input editor state
    agent: Option<AgentHandle>,       // running agent + channels
    session: SessionManager,
    config: Config,
    lua: LuaRuntime,
    model: Model,
    thinking_level: ThinkingLevel,
    status_items: HashMap<String, String>,  // footer status from extensions
    widgets: HashMap<String, WidgetContent>,
    tools_expanded: bool,
    mode: UiMode,  // Normal, TreeView, ModelSelector, CommandPalette
    accumulated_usage: Usage,
    accumulated_cost: Cost,
}
```

### Components

**Chat view** (`views/chat.rs`):
- Render message history with syntax highlighting (syntect)
- Streaming text: append each TextDelta, re-render the current message
- Thinking blocks: collapsible, different styling
- Tool calls: show tool name + args summary, expandable output (Ctrl+O)
- Auto-scroll to bottom on new content
- Markdown rendering: headers, bold, italic, code blocks, lists

**Editor** (`views/editor.rs`):
- Multi-line text input
- Enter to send, Shift+Enter for newline (Ctrl+J as fallback)
- @ triggers file fuzzy finder (use `nucleo` crate)
- Tab for path completion
- Border color indicates thinking level
- Ctrl+C clears, Ctrl+C twice quits, Escape aborts current operation

**Tool call rendering** (`views/tools.rs`):
- Compact view: tool name + args summary (one line)
- Expanded view: full tool output with syntax highlighting
- Toggle with Ctrl+O
- Streaming partial results during execution

**Status bar** (`views/status.rs`):
- Left: cwd (shortened), session name
- Right: total tokens (input↑ output↓), cost ($X.XX), context usage (%), model name
- Extension status items (from ctx.ui.set_status)

**Tree view** (`views/tree.rs`):
- Session tree navigator (activated by /tree or Esc Esc)
- Show branching structure
- Navigate with arrow keys, select with Enter
- Filter modes: all, no-tools, user-only

### Streaming text rendering in ratatui

This is the core technical challenge. ratatui is immediate-mode — you re-render the full visible area every frame.

**Approach:**
- Store full message text in `DisplayMessage.content: String`
- On each `TextDelta` event, append delta to content string
- Mark the message as dirty (needs re-render)
- Event loop: on any event (keyboard, agent, resize, or 33ms tick), re-render if dirty
- ratatui renders only the visible viewport (virtual scrolling) — even with a 50KB message, only ~40 lines are rendered per frame
- **Syntax highlighting cache:** Use `syntect` to highlight code blocks. Cache highlighted output per message. On delta, only re-highlight the last code block (or none if the delta is outside a code block). This avoids re-highlighting the entire message on every character.

**Scroll behavior:**
- Auto-scroll to bottom when new content arrives (user is at bottom)
- If user scrolled up, DON'T auto-scroll (they're reading history)
- Track scroll position as line offset from bottom
- Page Up/Down, mouse wheel for scrolling

### Message rendering detail

Each `DisplayMessage` has:
- role: user / assistant / tool_call / tool_result / system / compaction
- content: String (raw text)
- rendered: Option<Vec<StyledLine>> (cached rendered output)
- is_streaming: bool (still receiving deltas)
- tool_call_id: Option (links tool_call to tool_result)
- expanded: bool (for tool results)
- timestamp: u64

**User messages:** Render with a "You:" prefix in accent color. Show @file references.

**Assistant messages:** Render with markdown processing:
- Headers (# ## ###) → bold + size indicator
- **Bold** → ANSI bold
- *Italic* → ANSI italic
- `inline code` → different background/foreground color
- ```code blocks``` → syntax highlighting via `syntect` with language detection
- Lists (- and 1.) → proper indentation
- Links [text](url) → show text, make clickable via terminal hyperlinks if supported
- Use a lightweight markdown parser — `pulldown-cmark` for parsing, custom renderer for terminal output

**Thinking blocks:** Collapsible. Show "Thinking..." header, toggle with Ctrl+T. When expanded, show thinking text in muted/dim style.

**Tool calls:** Compact header line showing: tool icon + tool name + args summary
- read: `read src/main.rs`
- bash: `$ cargo test`
- edit: `edit src/lib.rs`
- write: `write src/new_file.rs`
- grep: `grep "pattern" src/`
Custom formatting per tool name for readability.

**Tool results:** Below the tool call header.
- Collapsed (default): one-line summary — "✓ 45 lines" or "✗ Error: file not found"
- Expanded (Ctrl+O): full output with syntax highlighting
- Streaming partial results: show "Running..." with partial output

**Compaction messages:** Show "[Context compacted: X tokens → Y tokens]" in dim style.

### Editor component detail

**State:**
```rust
pub struct EditorState {
    content: String,           // current text
```

**Key bindings:**
- Enter: send message (if content not empty)
- Shift+Enter / Ctrl+J: insert newline
- Up/Down: move cursor in multi-line; at first/last line, cycle history
- Left/Right: move cursor
- Ctrl+A: beginning of line
- Ctrl+E: end of line
- Ctrl+W: delete word backward
- Ctrl+U: delete to beginning of line
- Ctrl+K: delete to end of line

- Home/End: beginning/end of line

**@ file reference:**

- Walk project files (respect .gitignore via `ignore` crate)
- Show top 10 matches in a dropdown below the @ position
- Arrow keys to navigate, Enter to select, Escape to dismiss
- On select: insert the file path after @

**Path completion:**
- Tab triggers path completion
- Complete relative to cwd
- Show options if ambiguous

**Border color:** Changes based on thinking level:
- Off: default border color

### Model selector (/model or Ctrl+L)

- Replace editor area with a selection list
- Group by provider: Anthropic, OpenAI, Google
- Show model name, context window, pricing
- ✓ indicator for current model
- Fuzzy search as you type
- Enter to select, Escape to dismiss
- On select: switch model, dismiss, notify in footer

### Ctrl+P model cycling

- Cycle through "scoped models" (configurable subset)
- Default scoped models: sonnet, haiku, gpt-4o (configurable in settings)
- Show brief notification of new model in footer
- Shift+Ctrl+P cycles backward

### Session tree view (/tree or Esc Esc)

- Full-screen overlay showing the session tree
- Each node: one-line summary of the message (truncated)
- Visual tree structure with box-drawing characters
- Current position highlighted
- Arrow keys to navigate, Enter to select (jump to that point)
- Filter modes (Ctrl+O cycle): all → no-tools → user-only
- Search by typing
- `l` key to label/bookmark an entry

### Settings dialog (/settings)

- Overlay with toggle-able settings
- Thinking level: cycle through levels
- Auto-compaction: on/off
- Tool output expand: default expanded/collapsed
- Theme selection (when themes are available)
- Arrow keys + Enter to change, Escape to dismiss

### Slash command handling

When the editor content starts with `/`, show command autocomplete:
- List matching commands in a dropdown above the editor
- Commands: /model, /settings, /tree, /fork, /compact, /new, /resume, /session, /name, /copy, /export, /reload, /hotkeys, /quit, /login, /logout
- Plus skill commands (/skill:name) and prompt template commands (/template)
- Tab or Enter to select, continue typing to filter

### Message queue (steering + follow-up)

While the agent is streaming:
- Enter: queue a steering message (interrupt after current tool, delivered immediately)
- Alt+Enter: queue a follow-up message (delivered after agent finishes)
- Show queued messages indicator in editor border
- Escape: cancel current operation and restore queued messages to editor

### TuiInterface (implements UserInterface)

**notify(message, level):**
- Append a styled notification to the message area
- Info: accent color, Warning: yellow, Error: red
- Non-blocking, no user interaction

**confirm(title, message):**
- Show a bordered dialog in the editor area
- Two buttons: Yes / No (arrow keys or y/n keys)
- Return true/false
- Escape returns None (cancelled)

**select(title, options):**
- Show a selection list in the editor area
- Arrow keys to navigate, Enter to select
- Escape returns None
- If options have descriptions, show them below each label

**input(title, placeholder):**
- Show a text input in the editor area
- Enter to submit, Escape to cancel

**set_status(key, text):**
- Add/update/remove a status item in the footer
- Multiple extensions can have simultaneous status items

**set_widget(key, content):**
- Render string lines or a declarative component above the editor
- Multiple widgets stack vertically

**custom(component_spec):**
- Replace editor area with the declarative component
- Route key events to the component's on_key handler
- Re-render when state changes
- Return when done() is called
- Restore editor when complete
