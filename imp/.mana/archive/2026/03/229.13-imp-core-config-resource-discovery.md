TOML configuration and automatic discovery of AGENTS.md, skills, and prompt templates.

## Config system

### File: `crates/imp-core/src/config.rs`

**Config struct:**
```rust
pub struct Config {
    pub model: Option<String>,
    pub thinking: Option<ThinkingLevel>,
    pub max_turns: u32,
    pub tools: Option<Vec<String>>,
    pub roles: HashMap<String, Role>,
    pub hooks: Vec<HookDefinition>,
    pub context: ContextConfig,
}

pub struct ContextConfig {
    pub observation_mask_threshold: f64,  // default: 0.6
    pub compaction_threshold: f64,         // default: 0.8
    pub mask_window: usize,                // default: 10
}
```

**Resolution hierarchy** (highest wins):
1. CLI flags
2. Environment variables (IMP_MODEL, IMP_THINKING)
3. Project config (.imp/config.toml)
4. User config (~/.config/imp/config.toml)
5. Built-in defaults

**Config loading:**
- Parse TOML with the `toml` crate
- Merge configs layer by layer (project overrides user, user overrides defaults)
- Validate after merge (unknown keys warn, invalid values error)

## Resource discovery

### File: `crates/imp-core/src/resources.rs`

**AGENTS.md discovery:**
- Walk up from cwd to filesystem root (or git repo root)
- At each directory, check for AGENTS.md or CLAUDE.md
- Also check ~/.config/imp/AGENTS.md
- Concatenate all found files (global first, then root-to-cwd order)
- Return as a single string for system prompt injection

**Skills discovery:**
- Search directories:
  - ~/.config/imp/skills/*/SKILL.md
  - .imp/skills/*/SKILL.md (project)
  - Walk up from cwd: .agents/skills/*/SKILL.md
- Each skill: { name (directory name), description (first paragraph of SKILL.md), file_path, base_dir }
- Return Vec<Skill>

**Prompt templates:**
- Search directories:
  - ~/.config/imp/prompts/*.md
  - .imp/prompts/*.md
- Parse frontmatter (if any) for metadata (description, variables)
- Template expansion: replace {{variable}} with arguments
- Return Vec<PromptTemplate>

**Custom models:**
- Load from ~/.config/imp/models.toml
- Merge with built-in ModelRegistry
- Format: `[[models]]` array with provider, id, name, context_window, pricing, etc.

## Acceptance criteria

- Custom models merge with built-in registry
