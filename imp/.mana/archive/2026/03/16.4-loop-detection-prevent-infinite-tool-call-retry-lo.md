Prevent the agent from burning tokens in an infinite retry loop — calling the same failing tool with the same args repeatedly. This is a common failure mode where the model doesn't learn from tool errors.

## What to implement

1. **Add `LoopDetector` struct** in `imp-core/src/agent.rs` (or a new small module):
   ```rust
   pub struct LoopDetector {
       window: VecDeque<u64>,     // sliding window of hashes
       window_size: usize,        // default: 20
       max_repeats: usize,        // default: 5
   }
   ```
   - Hash signature: SHA-256 of `tool_name + serialized_args + is_error + first 200 chars of output`
   - Including the output prefix distinguishes identical calls with different results
   - `record()` adds a hash to the window, returns true if still OK
   - `is_looping()` returns true if any hash appears >= max_repeats times in the window

2. **Wire into agent loop**: after each tool execution (in `execute_tools`), record the result. Check after processing all tool results for a turn. If looping detected, emit `AgentEvent::Error` and break the loop.

3. **Add `loop_detector: LoopDetector` field to Agent**, initialized in `Agent::new`.

## Tests
- Feed identical (tool, args, result) tuples — verify detection triggers at threshold
- Feed varied calls — verify no false positive
- Verify detection resets after window slides past old entries
- Agent integration test: mock provider that always calls the same failing tool — verify loop breaks
