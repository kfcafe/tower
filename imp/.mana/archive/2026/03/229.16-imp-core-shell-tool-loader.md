Load tool definitions from TOML files. Zero-code CLI wrappers.

## What to implement

### File: `crates/imp-core/src/tools/shell.rs`

**ShellToolDef (parsed from TOML):**
```rust
pub struct ShellToolDef {
    pub name: String,
    pub label: String,
    pub description: String,
    pub readonly: bool,
    pub params: HashMap<String, ParamDef>,
    pub exec: ExecConfig,
    pub install_hint: Option<String>,
}

pub struct ParamDef {
    pub param_type: String,  // "string", "number", "boolean"
    pub description: String,
    pub optional: bool,
    pub default: Option<String>,
}

pub struct ExecConfig {
    pub command: String,
    pub args: Vec<String>,     // may contain {param} and {param|default} interpolations
    pub timeout: Option<u32>,
    pub truncate: TruncateMode,  // "head" or "tail"
    pub env: HashMap<String, String>,
}
```

**TOML format:**
```toml
name = "youtube_transcript"
label = "YouTube Transcript"
description = "Extract transcript from a YouTube video"
readonly = true
install_hint = "brew install yt-dlp"

[params]
url = { type = "string", description = "YouTube URL or video ID" }
language = { type = "string", description = "Language code", optional = true }

[exec]
command = "yt-dlp"
args = ["--write-subs", "--skip-download", "--sub-lang", "{language|en}", "{url}"]
timeout = 30
truncate = "head"
```

**Interpolation rules:**
- `{param}` — replaced with param value, error if missing and not optional
- `{param|default}` — replaced with param value, or default if missing
- Params are validated against the schema before interpolation

**ShellTool (implements Tool trait):**
- parameters() returns JSON schema generated from ParamDef map
- execute() runs the command via tokio::process::Command, captures stdout, applies truncation
- If command binary not found: return helpful error with install_hint if available

**Discovery:**
- Load from: ~/.config/imp/tools/*.toml, .imp/tools/*.toml, built-in tools/ directory
- Register all discovered shell tools in the ToolRegistry

**Built-in shell tools to ship:**
- youtube_transcript.toml (yt-dlp)
- pdf_read.toml (pdftotext from poppler)
- desktop_notify.toml (osascript on macOS)

## Acceptance criteria
- Parse a TOML tool definition correctly
- Interpolation works with params and defaults
- Missing required param → validation error
- Command execution produces correct output
- Missing binary → helpful error with install_hint
- Discovery loads from all paths
- Built-in tools parse correctly
