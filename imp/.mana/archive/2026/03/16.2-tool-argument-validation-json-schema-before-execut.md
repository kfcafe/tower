Every tool call from the LLM should be validated against the tool's JSON schema before execution. Currently args go straight to the tool — bad types, missing required fields, and extra fields aren't caught. Validation errors should be returned as tool results with `is_error: true` so the model can self-correct.

## What to implement

1. **Add `pub fn validate_tool_args`** in `imp-core/src/tools/mod.rs`:
   - Takes the tool's `parameters()` JSON schema and the args `Value`
   - Validates required fields present, types correct
   - Returns `Ok(())` or `Err(String)` with a human-readable error
   - Use the `jsonschema` crate (add to imp-core Cargo.toml)

2. **Wire into `execute_one_tool`** in `agent.rs`:
   - Before the hook check and tool execution, call `validate_tool_args`
   - If validation fails, return a ToolOutput::error with the validation message
   - Don't call the tool at all — save the execution cost

3. **Tests** in `tools/mod.rs`:
   - Test valid args pass validation
   - Test missing required field returns error
   - Test wrong type returns error  
   - Test extra/unknown fields are allowed (LLMs often add them)

## Key files
- `imp-core/src/tools/mod.rs` — Tool trait (has `parameters()` method), add validate_tool_args
- `imp-core/src/agent.rs` — execute_one_tool method, wire validation before execution
- `imp-core/Cargo.toml` — add jsonschema dependency
