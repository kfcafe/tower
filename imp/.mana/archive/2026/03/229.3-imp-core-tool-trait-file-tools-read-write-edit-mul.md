Define the Tool trait and implement the file-based tools. These are the most-used tools and need to be solid.

## What to implement

### File: `crates/imp-core/Cargo.toml`
Dependencies: imp-llm (path), serde, serde_json, tokio, async-trait, thiserror, anyhow, similar (for diff generation), walkdir, glob, tracing

### File: `crates/imp-core/src/tools/mod.rs`

**Tool trait:**
```rust
#[async_trait]
pub trait Tool: Send + Sync {
    fn name(&self) -> &str;
    fn label(&self) -> &str;
    fn description(&self) -> &str;
    fn parameters(&self) -> &serde_json::Value;
    fn is_readonly(&self) -> bool;
    async fn execute(&self, call_id: &str, params: serde_json::Value, ctx: ToolContext) -> Result<ToolOutput>;
}
```

**ToolContext:** { cwd: PathBuf, signal: CancellationToken, update_tx: Sender<ToolUpdate>, ui: Arc<dyn UserInterface> }

**ToolOutput:** { content: Vec<ContentBlock>, details: Value, is_error: bool }

**ToolRegistry:** HashMap of tools with register(), get(), definitions() methods.

**Truncation utilities:**
- `truncate_head(input, max_lines=2000, max_bytes=50000)` → TruncationResult
- `truncate_tail(input, max_lines, max_bytes)` → TruncationResult
- `truncate_line(line, max_bytes=500)` → String
- When truncated, write full output to temp file, include path in result

### Tool implementations:

**read.rs** — Read file contents:
- Params: { path: String, offset?: u32 (1-indexed line), limit?: u32 }
- Resolve path relative to cwd, strip leading @ (models sometimes include it)
- Detect binary files (check for null bytes in first 8KB), return error for binary
- Detect image files by extension (.png, .jpg, .gif, .webp, .svg), return ContentBlock::Image with base64
- Apply offset/limit for pagination (default: entire file)
- Apply truncation (head mode, 2000 lines / 50KB)
- If file not found: use `walkdir` to search for similar filenames, suggest top 3 by edit distance (Levenshtein)

**write.rs** — Write file contents:
- Params: { path: String, content: String }
- Create parent directories if they don't exist (tokio::fs::create_dir_all)
- Normalize line endings (detect existing, preserve; default to LF for new files)
- Return: file path, bytes written, whether file was created or overwritten

**edit.rs** — Find-and-replace with fuzzy matching:
- Params: { path: String, old_text: String, new_text: String }
- Read file, normalize to LF internally
- Exact match first (content.find(old_text))
- If no exact match, try fuzzy: strip trailing whitespace per line, normalize smart quotes/unicode dashes/special spaces to ASCII
- If still no match, return error with helpful message (show first few chars of file around where it might be)
- Apply replacement, restore original line endings
- Generate unified diff using `similar` crate
- Return: diff output, file path, whether fuzzy match was used

**multi_edit.rs** — Multiple edits to one file:
- Params: { path: String, edits: [{ old_text: String, new_text: String }] }
- Validate ALL edits find their targets before applying any (atomic)
- Apply edits sequentially (each edit sees the result of previous ones)
- If any edit fails to match, report which succeeded and which failed, apply nothing
- Generate combined diff
- Use the same fuzzy matching as edit.rs

**ls.rs** — List directory:
- Params: { path?: String, limit?: u32 }
- Default path: cwd
- Return sorted entries with / suffix for directories
- Include dotfiles
- Truncate to limit (default: 500 entries)
- Return: entries as text, one per line
