Implement the Anthropic Messages API provider with SSE streaming. This is the first working provider — the foundation for the agent loop.

## What to implement

### File: `crates/imp-llm/src/providers/anthropic.rs`

**Message serialization** — Convert imp's unified Message types to Anthropic's wire format:
- User messages: `{ role: "user", content: string | ContentBlock[] }`
- Assistant messages: `{ role: "assistant", content: ContentBlock[] }`
- Tool results: `{ role: "user", content: [{ type: "tool_result", tool_use_id, content }] }`
- Images: `{ type: "image", source: { type: "base64", media_type, data } }`
- Tool calls: `{ type: "tool_use", id, name, input }`
- Thinking blocks: `{ type: "thinking", thinking: "..." }`

**Cache control** — Anthropic uses `cache_control: { type: "ephemeral" }` breakpoints:
- On the last system prompt content block
- On the tool definitions block
- On the last N user messages (configurable via CacheOptions.cache_recent_turns)
- This is critical for cost savings on long sessions

**Thinking/reasoning** — Map ThinkingLevel to Anthropic's extended thinking:
- Off: don't include thinking parameter
- Minimal/Low/Medium/High/XHigh: set `thinking: { type: "enabled", budget_tokens: N }`
- Budget mapping: Minimal=1024, Low=4096, Medium=10000, High=32000, XHigh=100000
- When thinking is enabled, max_tokens must be > budget_tokens

**SSE streaming** — Parse Anthropic's server-sent events:
- Endpoint: POST `https://api.anthropic.com/v1/messages`
- Headers: `x-api-key`, `anthropic-version: 2023-06-01`, `content-type: application/json`
- Set `stream: true` in request body
- SSE event types to handle:
  - `message_start` → emit StreamEvent::MessageStart
  - `content_block_start` → track block type (text, tool_use, thinking)
  - `content_block_delta` → emit TextDelta, ThinkingDelta, or accumulate tool call JSON
  - `content_block_stop` → if tool_use block, emit StreamEvent::ToolCall with accumulated args
  - `message_delta` → extract stop_reason and usage
  - `message_stop` → emit StreamEvent::MessageEnd
  - `error` → emit StreamEvent::Error

**Tool definitions** — Convert ToolDefinition to Anthropic format:
- `{ name, description, input_schema: { type: "object", properties, required } }`

**Auth resolution** — Check AuthStore for API key:
- Priority: stored credential → ANTHROPIC_API_KEY env var → error
- Support both API key and OAuth token (same header: `x-api-key`)
