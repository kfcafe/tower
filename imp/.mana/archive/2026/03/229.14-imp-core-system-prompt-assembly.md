Assemble the system prompt from five layers. Minimal by design — no behavioral coaching.

## What to implement

### File: `crates/imp-core/src/system_prompt.rs`

**Five layers:**

**Layer 1: Identity (~200 tokens)**
```
You are imp, a coding agent.

Available tools:
- read: Read file contents. Supports text and images. ...
- write: Write content to a file. ...
- edit: Edit a file by replacing exact text. ...
(one line per active tool: name + description)

Guidelines:
- (tool-specific usage guidelines from tool definitions)
```

**Layer 2: Project context**
Concatenated AGENTS.md content from resource discovery. Verbatim, no transformation.

**Layer 3: Skills index**
```
Available skills (use read to load when relevant):
- rust: Conventions for writing Rust code. Use when working with .rs files. [path]
- testing: Write and review tests. [path]
```
One line per discovered skill: name, description, file path. The LLM reads the full skill content when it decides it's relevant.

**Layer 4: Mana facts** (if .mana/ exists)
```
Project facts:
- "This project uses JWT for auth" [verified 2h ago]
- "Test suite requires Docker running" [verified 1d ago]
```
All project facts from mana-core, with verification timestamps.

**Layer 5: Task context** (only in headless/task mode)
```
## Task
Title: Fix the failing auth test
Description: ...
Verify: cargo test auth::jwt_test

## Previous attempts
Attempt 1 (failed): Tried X, failed because Y
Attempt 2 (failed): Tried Z, got error W

## Dependencies
- Schema types (completed): defined in src/schema.rs
```

**Assembly function:**
```rust
pub fn assemble_system_prompt(
    tools: &ToolRegistry,
    agents_md: &str,
    skills: &[Skill],
    facts: &[Fact],        // from mana-core, empty if no .mana/
    task: Option<&TaskContext>,  // only in headless mode
    role: Option<&Role>,   // role-specific instructions
) -> String
```

**Role-aware assembly:**
- If a role has `instructions`, append them after Layer 1
- If a role is `readonly`, tool list only includes readonly tools

## Acceptance criteria
- Assembly produces correct output for each layer
- Missing layers handled gracefully (no .mana/ → skip Layer 4, no task → skip Layer 5)
- Role instructions appended correctly
- Tool descriptions pulled from registry
- Skills listed with descriptions and paths
- Total prompt size tracked (for cache control decisions)
