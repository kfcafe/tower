Event-driven hook system. TOML-configured hooks for simple shell actions, plus programmatic hooks for Lua extensions.

## What to implement

### File: `crates/imp-core/src/hooks.rs`

**HookEvent enum** — all events that can trigger hooks:
- AfterFileWrite { file: PathBuf }
- BeforeToolCall { tool_name: String, args: Value } — can return block=true
- AfterToolCall { tool_name: String, result: ToolResultMessage } — can modify result
- BeforeLlmCall
- OnContextThreshold { ratio: f64 }
- OnSessionStart
- OnSessionShutdown
- OnAgentStart { prompt: String }
- OnAgentEnd { messages: Vec<Message> }
- OnTurnEnd { index: u32, message: AssistantMessage }

**HookDefinition struct:**
```rust
pub struct HookDefinition {
    pub event: String,            // "after_file_write", "before_tool_call", etc.
    pub match_pattern: Option<String>,  // glob for files, tool name for tool events
    pub action: HookAction,
    pub blocking: bool,
    pub threshold: Option<f64>,   // for on_context_threshold
}
```

**HookAction enum:**
- Shell { command: String } — run shell command, interpolate {file}, {tool_name}
- BuiltIn(BuiltInAction) — ObservationMask, Compact
- Callback(Box<dyn Fn>) — for Lua hooks (registered programmatically)

**HookRegistry:**
- register(hook: HookDefinition) — add a hook
- register_callback(event, callback) — for Lua
- fire(event: HookEvent) -> Vec<HookResult> — execute all matching hooks
- Execution order: TOML hooks first (config order), then Lua hooks (load order)
- Blocking hooks execute sequentially and await completion
- Non-blocking hooks are spawned as tokio tasks

**HookResult:**
- block: bool (for BeforeToolCall)
- reason: Option<String>
- modified_content: Option<Vec<ContentBlock>> (for AfterToolCall)

**TOML loading:**
Parse hooks from config.toml:
```toml
[[hooks]]
event = "after_file_write"
match = "*.rs"
action = "shell"
command = "rustfmt {file}"
blocking = true
```

**Integration with agent loop:**
The agent loop calls `hook_registry.fire(event)` at each lifecycle point. For BeforeToolCall, check if any hook blocks. For AfterToolCall, apply any result modifications.

## Acceptance criteria
- TOML hook parsing works correctly
- Shell hooks execute with correct interpolation ({file} replaced with actual path)
- Blocking hooks block the caller, non-blocking hooks fire and forget
- BeforeToolCall hooks can block tool execution
- AfterToolCall hooks can modify tool results
- OnContextThreshold hooks fire at the correct ratio
- Hooks fire in correct order (TOML first, then programmatic)
