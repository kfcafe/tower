Replace the global Tab toggle with per-tool-call expand/collapse. Auto-expand tool calls that fail.

## Current behavior
- Tab toggles `tools_expanded` globally — ALL tool calls expand or collapse together
- No way to expand just one tool call
- Errors look the same as successes (just a red ✗ icon)

## New behavior

### Auto-expand errors
In `handle_agent_event`, when `AgentEvent::ToolExecutionEnd` fires with `is_error: true`:
- Set `tc.expanded = true` on that specific tool call
- Do NOT require global peek mode

In `app.rs`, find the `ToolExecutionEnd` handler (around line 1601):
```rust
AgentEvent::ToolExecutionEnd { tool_call_id, result } => {
    // ... existing output attachment code ...
    
    // Auto-expand errors
    for msg in self.messages.iter_mut().rev() {
        for tc in &mut msg.tool_calls {
            if tc.id == tool_call_id {
                tc.output = Some(output_text.clone());
                tc.is_error = is_error;
                if is_error {
                    tc.expanded = true;  // <-- ADD THIS
                }
                break;
            }
        }
    }
}
```

### Per-tool-call selection
Add to `App` struct:
```rust
pub tool_focus: Option<usize>,  // index into flattened tool call list across all messages
```

### Keybinding changes in `keybindings.rs`

Add new actions:

In `resolve_normal`:
- `Tab` → `Action::ToolToggle` (was Peek — now toggles focused tool, or all if no focus)

- Add: `Ctrl+Up` → `Action::ToolFocusPrev`

### Tab behavior (in handle_normal_key)
```rust
Some(Action::ToolToggle) => {
    if let Some(idx) = self.tool_focus {
        // Toggle just the focused tool call
        if let Some(tc) = self.get_tool_call_mut(idx) {
            tc.expanded = !tc.expanded;
        }
    } else {
        // No focus: toggle all (existing behavior)
        self.tools_expanded = !self.tools_expanded;
        for msg in &mut self.messages {
            for tc in &mut msg.tool_calls {
                tc.expanded = self.tools_expanded;
            }
        }
    }
}
```

### Tool focus navigation helpers in App
```rust
fn total_tool_calls(&self) -> usize {
    self.messages.iter().map(|m| m.tool_calls.len()).sum()
}

fn get_tool_call_mut(&mut self, flat_idx: usize) -> Option<&mut DisplayToolCall> {
    let mut remaining = flat_idx;
    for msg in &mut self.messages {
        if remaining < msg.tool_calls.len() {
            return Some(&mut msg.tool_calls[remaining]);
        }
        remaining -= msg.tool_calls.len();
    }
    None
}
```

### Visual indicator in chat.rs
When a tool call is focused (its flat index == `tool_focus`), render with a highlight:
- Add `focused: bool` parameter to `DisplayToolCall::header_line_animated()`
- When focused, use `theme.selected_style()` or prepend a `▸` indicator
- Pass `tool_focus` through `ChatView` (add field `tool_focus: Option<usize>`)

### Reset focus
- Reset `tool_focus = None` when sending a new message
- Reset when agent starts a new turn
