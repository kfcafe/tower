Improve the visual rendering of tool calls in the chat view so they're clearly distinct from assistant prose. Currently tool calls and text are interleaved with minimal visual separation.

## Current rendering (chat.rs)
Tool calls render as:
```
 ✓ read src/main.rs ✓ 42 lines
 ✓ edit src/main.rs ✓ 1 lines
 ✓ bash $ cargo test ✓ 15 lines
```

These look like content. They should look like *activity* — subordinate to the prose, visually grouped.

## New rendering

### 1. Group consecutive tool calls with a visual bracket

When multiple tool calls appear in sequence (common — agent often runs 3-5 tools between prose blocks), render them as a group:

```
  │ ✓ read src/main.rs                    42 lines
  │ ✓ edit src/main.rs                     1 lines  
  │ ✓ bash $ cargo test                   15 lines
```

The `│` left border creates a visual rail that groups activity together, distinct from prose.

### 2. Dimmed tool calls by default

Tool calls that succeeded should use `theme.muted` color for the args and line count, reserving full brightness for errors and the tool name.

### 3. Compact completed tools

Completed successful tool calls should be more compact:
- Tool name in bold magenta (existing)
- Args summary in dim (existing)  
- Result: just line count in dim on the right, NOT "✓ 42 lines"
- Remove the redundant ✓ before the line count (the ✓ icon already indicates success)

### Changes to `views/tools.rs`

Modify `header_line_animated()`:

```rust
pub fn header_line_animated(&self, theme: &Theme, tick: u64, focused: bool) -> Line<'static> {
    // ... existing icon logic ...
    
    let mut spans = vec![];
    
    // Left rail (for visual grouping in chat view — caller adds this)
    // Icon
    spans.push(Span::styled(format!(" {icon} "), icon_style));
    // Tool name
    spans.push(Span::styled(
        self.name.clone(),
        Style::default().fg(theme.tool_name).add_modifier(Modifier::BOLD),
    ));
    
    // Args in muted
    if !self.args_summary.is_empty() {
        spans.push(Span::raw(" "));
        spans.push(Span::styled(self.args_summary.clone(), theme.muted_style()));
    }
    
    // Result on the right — just line count, no redundant icon
    if !self.expanded {
        if let Some(ref output) = self.output {
            if self.is_error {
                spans.push(Span::styled(" error", theme.error_style()));
            } else {
                let line_count = output.lines().count();
                spans.push(Span::styled(
                    format!("  {line_count} lines"),
                    theme.muted_style(),
                ));
            }
        }
    }
    
    Line::from(spans)
}
```

In the tool call rendering section of `ChatView::render()`, add the left rail:

// Tool calls
for (i, tc) in msg.tool_calls.iter().enumerate() {
    let is_running = tc.output.is_none() && !tc.is_error;

    if is_running && !tc.streaming_lines.is_empty() {

        header.spans.insert(0, rail.clone());
