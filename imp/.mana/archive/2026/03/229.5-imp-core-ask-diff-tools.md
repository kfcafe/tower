Implement the ask and diff tools. Both depend on the Tool trait and UserInterface trait from 229.3.

## ask.rs — Ask user a question

Params: { question: String, context?: String, options?: (String | {label, description})[], multi_select?: bool, allow_other?: bool, default?: String|String[], placeholder?: String }

Implementation:
- If options provided: multiple choice (or multi-select)
- If options omitted: short text answer
- Call `ctx.ui.select()` for multiple choice, `ctx.ui.input()` for text
- If allow_other: add an "Other..." option that opens text input
- If ctx.ui.has_ui() is false: return error "Cannot ask user in this mode"
- Return: the user's answer as text

This tool works through the UserInterface abstraction — TUI mode renders widgets, headless mode sends JSON events through stdio, print mode returns an error.

## diff.rs — Diff tools (diff_show + diff_apply)

**diff_show** — Generate unified diff:
- Params: { file: String, new_content: String, context_lines?: u32 }
- Read current file content
- Generate unified diff using `similar` crate (TextDiff::from_lines)
- Format as standard unified diff with `---`/`+++` headers
- Default context: 3 lines
- Return: the unified diff text

**diff_apply** — Apply a unified diff:
- Params: { file: String, patch: String, dry_run?: bool }
- Parse the unified diff (handle `@@` hunks)
- For each hunk, find the target location in the file using context lines
- If context lines don't match exactly, try fuzzy matching (offset search within ±20 lines)
- If dry_run: return what would change without modifying the file
- If not dry_run: apply the patch and write the file
- Return: success/failure per hunk, what changed

Use the `similar` crate for diff generation. For patch application, implement a basic unified diff parser — it's straightforward: parse `@@` headers for line numbers, apply insertions (+) and deletions (-) relative to context.
