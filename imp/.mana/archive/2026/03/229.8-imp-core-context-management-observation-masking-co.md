**Stage 1: Observation masking** (triggered at ~60% context)
```rust
pub fn mask_observations(messages: &mut Vec<Message>, keep_recent_turns: usize)
```
- Count turns (a turn = one assistant message + its tool results)
- For turns older than `keep_recent_turns` (default: 10):
  - Replace ToolResultMessage content with a placeholder:
    `"[Output omitted — ran {tool_name}({summary_of_args}), returned {byte_count}]"`
  - Summary of args: first 100 chars of JSON-serialized args
  - Preserve: tool_call_id, tool_name, is_error flag
- This is purely in-memory manipulation of the messages Vec — doesn't affect the session file
- The original data is still in the session file for `/tree` navigation

**Stage 2: LLM compaction** (triggered at ~80% context)
```rust
pub async fn compact(
    messages: &[Message],
    model: &Model,
    options: CompactionOptions,
) -> Result<CompactionResult>

pub struct CompactionOptions {
    pub custom_instructions: Option<String>,
    pub keep_recent_turns: usize,  // default: 3
}

pub struct CompactionResult {
    pub summary: String,
    pub first_kept_id: String,
    pub tokens_before: u32,
    pub tokens_after: u32,
}
```

- Select messages to summarize: everything before the last `keep_recent_turns` turns
- Build a compaction prompt that instructs the model to preserve:
  - The user's original request (verbatim, quoted)
  - The current goal and what's been accomplished
  - Files read, written, and edited (list paths)
  - Tests added, commands run
  - Work remaining: next steps, blockers
  - Things that were tried and FAILED (critical for avoiding loops)
  - Things explicitly forbidden by the user
- Send to the model (using the same provider as the agent)
- Return the summary text
- The agent loop replaces old messages with a single Compaction message containing the summary

**Integration with agent loop:**
Add a `check_context_budget()` method to Agent that:
1. Calculates context_usage
2. At 60%: call mask_observations
3. At 80%: call compact, emit CompactionStart/CompactionEnd events, replace messages

## Acceptance criteria
- Token estimation: verify rough accuracy (within 2x of actual for English text)
- Observation masking: 20 turns → mask first 10 → verify only last 10 have full tool output
- Masking preserves: user messages, assistant text, tool call args (but not tool results)
- Compaction: mock provider returns a summary → verify old messages replaced with summary + recent turns kept
- Context usage calculation works correctly with masked vs unmasked messages
