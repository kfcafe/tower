## Task

Add `context_prefill` to `SessionOptions` and inject the prefill messages into the agent's conversation before the first prompt.

## Implementation

### 1. Add field to SessionOptions

In `crates/imp-core/src/imp_session.rs`, add to `SessionOptions`:

```rust
/// Pre-assembled context messages injected before the first prompt.
/// These appear as early conversation history so the cached prefix
/// includes the file context on every subsequent turn.
pub context_prefill: Vec<imp_llm::Message>,
```

Default: `Vec::new()` (empty = no prefill, zero overhead).

### 2. Inject in ImpSession::prompt()

In the `prompt()` method, after loading history from session but before spawning the agent task, prepend the prefill messages:

```rust
// Inject context prefill (once, on first prompt only)
if !self.context_prefill_injected {
    for msg in &self.context_prefill {
        agent.messages.insert(prefill_insert_idx, msg.clone());
    }
    self.context_prefill_injected = true;
}
```

The prefill messages go at the START of the conversation (before any session history), because:
- They form the cacheable prefix (must be first)
- Session history follows
- The current prompt is appended last by Agent::run()

Add `context_prefill: Vec<imp_llm::Message>` and `context_prefill_injected: bool` fields to `ImpSession`.

### 3. Thread through from SessionOptions to ImpSession

In `ImpSession::create()`:
```rust
Ok(Self {
    // ... existing fields ...
    context_prefill: options.context_prefill.clone(),
    context_prefill_injected: false,
})
```

Also add a prefill assistant response after the context user message so the conversation alternates correctly (user/assistant/user):

```rust
if !self.context_prefill.is_empty() && !self.context_prefill_injected {
    for msg in &self.context_prefill {
        agent.messages.push(msg.clone());
    }
    // Add assistant acknowledgment to maintain user/assistant alternation
    agent.messages.push(imp_llm::Message::Assistant(imp_llm::AssistantMessage {
        content: vec![imp_llm::ContentBlock::Text {
            text: "Context loaded. Ready to work.".into(),
        }],
        usage: None,
        stop_reason: imp_llm::StopReason::EndTurn,
        timestamp: imp_llm::now(),
    }));
    self.context_prefill_injected = true;
}
```

## Don't
- Don't change Agent::run() — the prefill is just pre-populated history
- Don't persist prefill messages to session storage — they're ephemeral context
- Don't add cache_control here — that's handled by the existing cache_recent_turns in imp-llm
