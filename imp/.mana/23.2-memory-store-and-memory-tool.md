Implement the MemoryStore struct and MemoryTool for agent-curated persistent memory.

## What to build

### 1. MemoryStore (`imp-core/src/memory.rs`)

A persistent store backed by a markdown file. Entries separated by `\n§\n`.

```rust
pub struct MemoryStore {
    path: PathBuf,
    entries: Vec<String>,
    char_limit: usize,
}
```

Methods:
- `load(path, char_limit)` — Parse the file, splitting on `§` separator. Create file if missing.
- `save()` — Write entries back, joined by `\n§\n`.
- `add(content)` — Append entry. Error if would exceed char_limit. Reject exact duplicates.
- `replace(old_text, content)` — Find the unique entry containing `old_text` substring, replace with `content`. Error if 0 or 2+ matches.
- `remove(old_text)` — Find unique entry containing `old_text`, remove it. Error if 0 or 2+ matches.
- `entries()` — Return slice of entries.
- `usage()` — Return `(used_chars, limit)`.
- `render(label)` — Render for system prompt injection with header showing usage %.

Security: Block entries with prompt injection markers (`<system>`, `[INST]`, `<<SYS>>`) or invisible Unicode.

### 2. MemoryTool (`imp-core/src/tools/memory.rs`)

Implement the `Tool` trait. Parameters: action (add|replace|remove), target (memory|user), content, old_text.

Paths: `memory` → `~/.config/imp/memory.md`, `user` → `~/.config/imp/user.md`

Returns JSON with success, message, entries, usage.

### 3. Wire into mod.rs

Add `pub mod memory;` to both `imp-core/src/tools/mod.rs` and `imp-core/src/lib.rs`.

### 4. Tests (in memory.rs #[cfg(test)])

- Load/save round-trip
- Add entries, verify capacity enforcement
- Replace with unique/ambiguous substring

- Duplicate detection
- Security scanning
