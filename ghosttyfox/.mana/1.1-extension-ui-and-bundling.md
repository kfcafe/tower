Research/context:
- `ghostty-web` exports `Ghostty`, `Terminal`, and `FitAddon`.
- initialization pattern should be:
```javascript
import { Ghostty, Terminal, FitAddon } from "ghostty-web";
const ghostty = await Ghostty.load(browser.runtime.getURL("dist/ghostty-vt.wasm"));
const term = new Terminal({ ghostty, cols: 80, rows: 24, cursorBlink: true, fontSize: 14, scrollback: 10000 });
const fit = new FitAddon();
term.loadAddon(fit);
term.open(container);
fit.fit();
fit.observeResize();
```
- `Terminal` supports `open`, `write`, `onData`, `onResize`, `loadAddon`, `resize`, `cols`, and `rows`.
- output from the host should be written as `Uint8Array` when possible.

Steps:

1. Create `extension/manifest.json`.
   Use Firefox Manifest V2 with this shape:
   ```json
   {
     "manifest_version": 2,
     "name": "Ghosttyfox",
     "version": "0.1.0",
     "description": "Run a real terminal in a Firefox tab using Ghostty's WASM terminal engine.",
     "permissions": ["nativeMessaging", "tabs"],
     "browser_specific_settings": {
       "gecko": {
         "id": "ghosttyfox@tower.dev"
       }
     },
     "browser_action": {
       "default_title": "Open Ghosttyfox"
     },
     "background": {
       "scripts": ["background.js"]
     },
     "content_security_policy": "script-src 'self' 'wasm-eval'; object-src 'self'",
     "web_accessible_resources": ["dist/*"]
   }
   ```

2. Create `extension/background.js`.
   Keep it tiny:
   ```javascript
   browser.browserAction.onClicked.addListener(() => {
     browser.tabs.create({
       url: browser.runtime.getURL("terminal.html")
     });
   });
   ```

3. Create `extension/terminal.html`.
   Requirements:
   - minimal HTML shell
   - link `terminal.css`
   - contain `<div id="terminal"></div>`
   - load `dist/terminal.bundle.js`
   - no inline JS

   Use this structure:
   ```html
   <!doctype html>
   <html lang="en">
     <head>
       <meta charset="utf-8" />
       <meta name="viewport" content="width=device-width, initial-scale=1" />
       <title>Ghosttyfox</title>
       <link rel="stylesheet" href="terminal.css" />
     </head>
     <body>
       <div id="terminal"></div>
       <script type="module" src="dist/terminal.bundle.js"></script>
     </body>
   </html>
   ```

4. Create `extension/terminal.css`.
   Keep it minimal and full-screen:
   ```css
   html, body {
     margin: 0;
     padding: 0;
     width: 100%;
     height: 100%;
     overflow: hidden;
     background: #111318;
   }

   body {
     font-family: ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace;
   }

   #terminal {
     width: 100%;
     height: 100%;
   }
   ```

5. Create `extension/terminal.js` as the extension runtime.
   It must:
   - load Ghostty WASM via `Ghostty.load(browser.runtime.getURL("dist/ghostty-vt.wasm"))`
   - create `Terminal` with clean defaults
   - load `FitAddon`
   - open the terminal into `#terminal`
   - call `fit.fit()` and `fit.observeResize()`
   - connect to the native host with `browser.runtime.connectNative("ghosttyfox")`
   - send `{ type: "input", data }` on user input
   - send `{ type: "resize", cols, rows }` on resize
   - decode base64 host output and write it back to the terminal
   - print clear terminal-visible status messages on `exit`, `error`, and disconnect

   Use this message model:
   ```javascript
   port.postMessage({ type: "input", data })
   port.postMessage({ type: "resize", cols, rows })
   ```
   and receive:
   ```javascript
   { type: "output", data: "<base64>" }
   { type: "exit", code: 0 }
   { type: "error", message: "..." }
   ```

   Implementation details:
   - use `Uint8Array.from(atob(...), c => c.charCodeAt(0))` to decode base64 output
   - call `term.write(binary)` for output
   - after the initial `fit.fit()`, send one initial resize message
   - keep the file direct and readable; no extra abstraction layers

6. Create `scripts/build.js`.
   Responsibilities:
   - ensure `extension/dist/` exists
   - bundle `extension/terminal.js` to `extension/dist/terminal.bundle.js` with esbuild
   - use an extension-safe browser target and ESM output
   - copy `node_modules/ghostty-web/ghostty-vt.wasm` into `extension/dist/ghostty-vt.wasm`

   Keep it as a small Node script using `esbuild`, `fs`, and `path`.
