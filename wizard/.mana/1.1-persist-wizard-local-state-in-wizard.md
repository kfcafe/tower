## What to implement
Turn `wizard-store` from a path helper into a real local-state persistence layer for Wizard.

## Files
- `crates/wizard-store/src/lib.rs`
- optionally tests inside the same crate

## Context
Wizard's config and state boundaries are now defined:
- `.mana/` is shared work state
- `.wizard/` is local user state
- `.wizard.toml` / `~/.config/wizard/config.toml` are config surfaces

This unit is only about local `.wizard/` state. It should load and save `state.json`, create parent directories when needed, and keep the API small and explicit.

Current code only defines `WizardLocalState` and `state_path()`.

## Acceptance criteria
- `wizard-store` can load local state from `.wizard/state.json`
- `wizard-store` can save local state back to disk
- missing state file is handled gracefully
- API stays focused on local state, not shared project policy
