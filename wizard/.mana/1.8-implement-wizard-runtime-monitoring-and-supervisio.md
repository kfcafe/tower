## What to implement
Build Wizard's first real runtime monitoring and supervision surface on top of the read-only backbone.

## Files
- `crates/wizard-orch/src/lib.rs`
- `crates/wizard-proto/src/lib.rs`
- desktop files under `app/desktop/src/` as needed

## Context
Once Wizard can load snapshots and render a basic shell, the next step is showing live agent/runtime state: running work, queued work, and process lifecycle updates. This should remain orchestrator-owned, not pushed into local UI state.

This is intentionally a follow-up phase and may need further decomposition before implementation.

## Acceptance criteria
- Wizard exposes runtime monitoring data beyond static snapshots
- desktop and backend have a typed surface for runtime updates
- orchestration state remains separate from `.wizard/` local UI state
